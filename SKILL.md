# TikTok 带货短视频策划 (Product Selling Video Planner)

把一个产品（图片或描述）+ 一个目标市场，变成一套**可直接拿去生产**的带货短视频方案。重点是**分镜提示词能直接复制进 AI 视频工具且不翻车**，文案能直接贴上平台。

用户用什么语言交流，就用什么语言跟他沟通；但**面向投放的文案要用目标市场的语言**（见第 4 步）。

## 两种用法（先判断走哪条）

- **模式 A · 产品 → 视频方案（默认）**：用户给产品/卖点，要做带货视频。走本文件主流程（第 0–5 步 + 后期剪辑）。
- **模式 B · 参考视频 → 拆解 + 仿写**：用户给了一条参考/竞品/爆款视频，想拆解结构、做"二次拆解"或仿拍/仿写模板。**读 `references/video-deconstruction.md` 按它执行**；脚本 `scripts/video_deconstruct.py` 做预处理（抽帧/网格/抽音频/转写）。拆出的「仿写模板」再回到模式 A 套用户的产品。

两种模式共用下面的「分镜提示词」「本地化文案」「后期剪辑」方法。

## 你会产出什么（5 个模块）

1. **痒点 / 卖点分析** —— 提炼这个产品最戳人的那个冲突点 + 一个主打利益点
2. **脚本结构表** —— 默认 15 秒，逐秒分配「画面 × 素材来源」
3. **分镜提示词** —— **默认出两套：即梦版 + Veo 版**，每条整段可复制，内置防翻车方法论
4. **本地化文案** —— 目标市场语言的标题 / 字幕 / CTA / 话题标签
5. **合规红线** —— 轻量提醒，避免限流下架

完整产出按文末「输出模板」组织。

## 核心原则（先理解，再生成）

**AI 视频模型不理解"因果累积"。** 它把提示词里所有描述都当成"从第 1 帧就存在的属性"去渲染——你写"梳出一堆毛""倒出满杯泡沫""吸出大量黑头"，它不会理解成"动作发生→之后才产生效果"，而是直接在开头就把那堆东西糊上去，画面立刻崩。

但有个关键特性能救场：**AI 不会"边动边生成效果"，却很会画"静态的结果"。** 一张"梳齿上挂着一撮毛"的静图，AI 画得又快又真；难的只是"视频里毛被一点点拔出来"。

所以本技能的主线是：**把控制权从模型手里抢回来**（用图锁住起点、用静图呈现结果），而不是用更长的提示词去"求"它。这条原则在第 3 步全面展开，且**默认全程在 AI 工具内完成，不需要实拍条件**。

> **根因框架（出任何商品提示词前先读）**：所有翻车都是同一根因——**AI 用自己的猜测填补你没锁死的每一个维度**。需锁的维度有六个：①身份外形 ②空间（地图/方向/光/此处为空）③时间（每拍多久）④状态突变（藏 or 切，插值会摊平突变）⑤过程动作（写"怎么到达"不只写到达状态）⑥镜头机位。完整框架 + 跨品类示例 + 开拍前自检清单见 `references/controlled-generation-framework.md`。剧情类/演绎类项目尤其必读。

## 第 0 步：收集输入

**🔴 CHECKPOINT · 🛑 STOP — 先确认，再出全量（流程铁规，覆盖一切出提示词请求，无例外）**：接到**任何**"出提示词/帮我生成"的请求——**不只是带货视频，也包括单张出图、写真、海报、场景/仙境大片等一切**——**第一步永远是先确认，绝不直接开始写分镜/出全量**。常犯错：把"出图/出单条提示词"当成不用问的例外，闷头就出分镜——没有例外。
正确顺序：①先用 AskUserQuestion 对齐**形态(图/视频/先图后视频)、数量(单张/一套)、用途、画幅**（视频还要：创意方向/主场景/素材路线/时长/钩子/生成路线）；②确认后先给**简短创意骨架/单条说明**让用户点头；③才展开全部逐镜头提示词和剪辑表。先写全量 = 方向一错全作废。每次准备"写分镜"前自检一句：**形态和用途我确认过了吗？** 收到反馈就回骨架层调，不在全量稿上缝补。

> **⚠️ 剧情/演绎类的②骨架确认默认直接出"可视化故事板"，不要用纯文字概要表**：只要角色在画面里有空间移动（抢占/奔赴/对峙/追逐/多角色同框），骨架确认环节就直接产出「带标注的可视化故事板 + 动线审计」（见 3.3 规矩 5）。文字概要表锁不住朝向 / 入画出画 / 镜头路径，正是这类片最易翻车处（动线打架、瞬移、凭空多出第二只主体），用文字表对齐 = 在错误的抽象层确认。**别等用户提醒才上故事板。** 非剧情类（单人口播、纯产品展示）才用文字骨架表。

开工前确认这几项；缺了关键项就用合理默认值并说明，只有当真的会改变方案时才追问。

| 输入 | 说明 | 默认 |
|------|------|------|
| 产品 | 图片或文字描述。**给了图就先读图**，提取材质、颜色、卖点细节、尺寸 | 必需 |
| 目标市场 | 决定文案语言与审美（如巴西=巴葡、日本=日语） | 缺了就问，或先按用户所在地 |
| 平台 | TikTok / Reels / 抖音 / 视频号 | TikTok |
| 时长 | 影响脚本节奏与镜头数 | 15 秒 |
| 卖点角度 | 用户若指定主打点就照做 | 由你从产品提炼 |
| 视频工具 | 即梦 / 可灵 / Runway / Sora / Veo | 即梦 |
| 实拍条件 | 有没有真机/真场景可拍 | **默认无 → 走全 AI 路线** |

**读产品图时**：把图里能看到的差异化卖点都记下来（如"双排刀片""浮动刀头""防滑纹理""不锈钢""尺寸标注"），后面分镜和文案要用。

## 🔴 先选生成路线（决策闸 · 🛑 STOP 等用户选 · 每条视频开工第一道决策，按题材适配选，不按省力）

出任何分镜前，**必须用 AskUserQuestion 弹出"两条路线"选项让用户选，并标出我的推荐和理由**——不替用户默默拍板，也别惯性上逐镜。选项这样给：

- **A · 整片一次出（故事板演绎）**：一张故事板大图喂 Seedance 一次演绎出整段。**快、便宜、多主体/连贯一致性强**；代价=运镜/节奏/单镜时长由模型定、且**含变身等突变要特殊处理**。
- **B · 逐镜分镜（逐镜+剪辑）**：每镜单独生成再拼。**精确控制每一拍(运镜/卡点/慢镜头/产品硬特写)**；代价=慢、贵、多镜人物一致性要靠角色卡。

**按题材给推荐**（弹选项时一并说"我建议 X，因为…"）：
| 题材特征 | 推荐 | 为什么 |
|---|---|---|
| 多主体/多角色一致性（多猫/多人/群像）、连续动作剧情、短剧·搞笑反转段子（时间轴脚本）、萌宠日常、运动一镜到底、走秀 | **A 整片演绎** | 一次生成天然统一，甜区，快且便宜 |
| 产品硬细节英雄特写、要精确卡点/运镜/慢镜头气场 | **B 逐镜** | 文字级精确控制 |
| 含状态突变（变身/换装/前后对比） | **看主体**：单主体→B逐镜+黑场最稳；多主体→A演绎但突变用遮挡藏；**带货前后对比→A演绎也可：中间插一格产品特写当切口藏换装（实测成立，详见 seedance2 档案）** | 演绎插值会摊平突变 |
| 单人口播 / 纯产品展示 | 看精度需求，推荐里说明 | — |

⚠️ 自检：推荐是按题材适配给的，还是套了"逐镜=保险"的惯性？控制点随路线变——逐镜靠文字写细、演绎靠故事板图画细文字精简。**用户选定后再继续，不替用户跳过这一步。**

> **⛔ 选定路线后，先去读该路线的硬要求再动手，别凭逐镜旧习惯或记忆开干（实测踩坑：选 A 演绎却套逐镜的"单帧+长文字"法、还漏给秒数，被连纠两次）**：A 演绎读 `references/seedance2-multimodal.md`（控制在图不在文字 / 多格故事板大图 / 每格秒数 / 强制竖版 9:16 等硬要求都在那份，此处不重复），剧情类加 `controlled-generation-framework.md`。两条路线控制点相反，用错范式=最典型翻车。另：别把没实测的模型限制当事实去劝退用户——给最佳版让用户去抽，回片抽帧验收。

> **本用户全局偏好（默认遵守）**：转场**优先无缝衔接**（尾帧承接/遮挡切/白闪掩护），不喜欢生硬硬切——硬切只在快剪节奏确需时用。机制见 3.3 切点。

## 🔴 先定画风轨道（决策闸）：原生直拍流 vs 棚拍广告流

出分镜之前先二选一（也可分镜混用：产品特写走棚拍、人物场景走原生），它决定每条提示词的风格词怎么写：

- **原生直拍流（C 端种草类默认）**：信任感来自"像真人博主随手拍"。提示词风格词用：**手机直拍质感、手持轻微晃动、自然窗光 / 生活光源、真实生活场景带生活痕迹、抓拍感、轻微不完美、深景深（画面整体清晰）、轻微噪点**；并**删掉**"电影级布光、摄影棚、4K 广告质感、浅景深虚化"这类词——手机镜头没有电影级虚化，背景一虚就露出广告感，种草视频一旦像品牌 TVC，划走率反升。
  镜头参数直接用 iPhone 焦段语言写，比泛写"手机感"更可控：**24mm（1x，日常场景）/ 77mm 微距（3x，皮肤·材质·产品细节特写）/ 超广角带轻微边缘畸变（自拍·收尾举产品，畸变一出就是"前置随手拍"的味道）**。
  **抖音"二压糊"质感（要画面更像真发布的 UGC、换装/穿搭等真人种草尤其有效，叠在"手机直拍"之上）**：写"降低锐度、不要高清锐利、带短视频平台二次压缩的轻微模糊与画质损失、轻微运动模糊拖影、自动曝光与白平衡自然漂移、适度美颜柔肤但保留真实皮肤光泽与轻微噪点、偏柔略'糊'像手机拍完直发抖音"，并写"全程 real-time 1 倍速、严禁升格慢放"。⚠️ 反直觉但关键：**故意降画质反而增真实**（电影级超清=广告感=划走），是"轻微噪点找 UGC 感"的加强版。
  **镜面自拍视角（穿戴/美妆类高代入机位）**：写"镜头就是镜子，第一人称镜面视角，观众即那面镜子"——人物对着镜头（=镜子）整理、试穿、展示，代入感极强且天然解决全身展示构图。配"自然手持微抖增强真实感"。
  **低角度上仰机位（服装类显身材比例）**：相机略微上仰拍模特全身，拉长腿部、强化剪影——和镜面视角互补（镜面=代入感，低角=显比例）。
  **"伪直播截图"风格（蹭赛事/热点的氛围带货，自带抓拍可信度）**：把画面做成"电视直播被暂停的一帧"——`电视直播色彩分级 + 轻微 MPEG 压缩噪点 + 隐约隔行扫描线 + 长焦压缩 + 暂停的 1080i 电视帧感 + 左上角比分条 + 角落网络台标`。用于看台名流同款服装/配饰带货等。要点：**故意加技术瑕疵（压缩噪点/隔行线）反而增真实感**（同"轻微噪点找 UGC 感"，这是电视质感版）；比分条文字用大写短词+数字（AI 能写对，见文字判据）。⚠️ **赛事/品牌/队名/台标全部用虚构替代**（如瞎编 MET/HBR 队名、虚构台标），既要真实感又避开真实 IP 侵权——这个"虚构替代真实 IP"手法通用于一切蹭热点/蹭场景的带货素材。
  **目录式拆解图版式（服装/配饰/套装带货的主图/详情图/展示帧）**：一张图=左侧分区编号清单（每件单品白底抠图+"1x"数量标）+ 右侧模特全身上身图 + 顶部大字品名。把"这套由哪几件组成 + 上身效果"一图讲清。⚠️ 图里的文字（品名/清单标签/数量）AI 生成必乱码——**文字一律后期排版加**，AI 只出"模特+单品抠图"的画面，或把 AI 图当草版、文字层后期重做。
  **逐层揭示/换装展示（服装带货钩子）**：拉开外套露出内搭、脱下罩衫现出主推单品、一衣多穿切换——"揭示下一层"本身是强钩子（观众想看里面是什么）。薄纱/蕾丝/真丝等需要展示材质透感的，正常写"薄如蝉翼、半透薄纱、垂坠真丝"等面料词以呈现质感。这些是服装/内衣品类的正常展示技法，放心用。
  **"穿上它人就变了"情绪反转钩子（服装/美妆爽点结构）**：不演产品多好，演**换上单品瞬间状态突变**——从慵懒/平淡 → 瞬间自信发光，镜头微推捕捉眼神变化。情绪转变即卖点，比平铺直叙展示强。⚠️ 这是"状态突变"，按需用遮挡/快切处理（见框架维度4）。
- **棚拍广告流（B2B / 工业 / 科技设备类默认）**：信任感来自"正规厂家"。用摄影棚布光、深色渐变背景、产品广告质感；这类买家看的是实力背书，刻意的"原生感"反而显小作坊。
- **判断**：卖给个人消费者的冲动型品类 → 原生；卖给老板 / 采购的工具设备 → 棚拍；拿不准就问用户要"博主随手拍"还是"官方产品片"的感觉。

## 第 1 步：痒点 / 卖点分析

带货视频的前 2 秒决定生死，而钩子来自**痒点**，不是卖点。
> 痒点是钩子的**内容层**（戳什么）；前 2 秒**怎么呈现**才抓眼是**形式层**——12 种钩子心理学模式（打破现实/好奇缺口/多巴胺/原始触发）+ 钩子运镜/音效 timing 见 `references/hook-library.md`，别每条都默认"痒点冲突"一种形式。

- **找冲突**：用户用这个产品前，最痛 / 最尴尬 / 最烦的瞬间是什么？（例：宠物毛打结、厨房油污难擦、黑头反复）
- **选一个主打利益点**：一条 15 秒视频只讲清**一个**好处，贪多必糊。其余卖点丢给文案或后续视频。
- **可代入**：痒点画面要让目标观众"这就是我家 / 我"，而不是抽象功效。

输出：1 句痒点 + 1 个主打利益点 + 1 句话讲清"为谁解决什么"。

> **想梗/想故事创意时**：别空想（不稳）。先查 `references/video-deconstruction.md` 的「🎭 梗结构库」——挑 1-2 个已验证的梗结构（拟人/反差变身/痛点救场/格式挪用/奇观化…）往产品上套。要新鲜/本地化的梗，就拆一条当下爆款补进库。梗的可靠来源是"拆真实爆款"，不是拍脑袋。

## 第 2 步：脚本结构（15 秒黄金结构，可调）

逐秒排「画面 × 素材来源」。**前 2 秒必须出痒点冲突**，否则刷走。

| 时间 | 作用 | 画面 | 素材来源 |
|------|------|------|---------|
| 0–2s | 钩子 | 痒点 / 冲突特写 | AI |
| 2–4s | 引入 | 产品登场、准备使用 | AI |
| 4–9s | 核心爽点 | 使用过程 / 效果证明（**最重要**） | AI（用第 3 步铁律 5 呈现效果）|
| 9–12s | 转折 | 使用后的对比 / 满意反应 | AI |
| 12–15s | 收口 | 产品展示 + 价格 / 优惠 + CTA | AI |

**镜头数量（重要，别一上来堆太多段）**：AI 每段都要单独生成 + 挑不崩的，段越多越累。
- **≤15s：默认 2–3 段**，4 段是上限（想要更碎节奏或更强控制时才用）。
- 上面 5 个"作用"不等于 5 段视频——可以合并。例如痒点+引入合成 1 段、顺滑+产品+CTA 合成 1 段。
- "效果结果特写"（铁律 5）和"白底产品"本来就是**硬切插入**，不算"需要无缝的叙事段"。
- 节奏靠剪辑卡点，不必靠多生成几段来加快——一段里也能推近 / 平移制造变化。
- **每段生成时长：最少 4s，建议 4–10s**（多数 AI 视频模型不接受 3.5s 这种）。脚本按整秒规划，直接按需要的时长生成，6s 就写 6s，4s 就写 4s；**不要设计"生成 4s 再裁短"的镜头**——浪费生成次数，且裁剪后节奏未必对。超过 10s 单镜头主体稳定性下降，宁可拆两段再硬切。

**调时长**：缩到 9s 就砍「转折」；放到 30s 就在「核心爽点」加多一个卖点演示。**核心爽点段永远不能省。**

**完播率 + 结尾可循环（平台算法 craft）**：完播率是 TikTok/Reels 最重的权重之一——80%+ 完播能拿约 2–3x 推流。所以①钩子前置（前 2 秒，已在做）②宁短勿长（能 12–15s 讲完别拖）③**收尾帧尽量能接回开头首帧**（无缝循环，观众多刷一遍＝完播率直接翻倍）。

> **做完整长广告（30–120s，多段拼接、突破单次 15s 上限）→ `references/long-form-video.md`**：资产前置（人物/场景/产品/音色统一）、三种分镜生成法、尾帧衔接踩坑、旁白整段生成等长视频专属流程。本主流程只管 ≤15s 短片。

**⭐ 前后对比类：把"产品展示"放在痛点和爽点中间，坐实"因为它才变好"的因果（实测·叙事经验）**：痛点对比片别把产品展示堆在结尾——结尾展示只是"顺便给你看看货"，说服力弱。把**产品 showcase 放在 痛点→产品→爽点 的正中间**当因果枢纽，观众潜意识就把"变强 / 变好"归功于这个产品（问题 → 产品(因) → 结果(果)，因果咬死）。额外红利：A 整片演绎路线里，**中间这格产品镜还兼着"藏换装突变"的切口功能**（见 seedance2「产品插入格」破解法）——一举两得：既是卖点展示=因，又是换装的场景切换点。结尾只留短 CTA / 系列色快闪即可。

**痛点段的"无效努力"节拍（可选但强烈推荐）**：痛点不只展示惨状，再加一拍**"老办法用力试了也没用"**（毛巾使劲搓还是糙、普通梳子越梳越乱）——情绪砸得更深，产品救场的反差才够大。配套检验标准：**痛点够丑、爽点够爽**——痛点镜和爽点镜并排对比，反差强度就是整条片的说服力；痛点画得太体面，后面全白搭。

## 第 3 步：分镜提示词（本技能的核心）

### 3.1 通用设置

- **画幅 9:16**（竖屏）。提醒用户在工具里选对，横版裁竖会切掉主体。
- **打开角色 / 主体一致性**（即梦故事板、可灵等都有），让产品和主角在多镜头间不变样。
- **参考图（首帧 / 首尾帧 / 结果图）用图像模型生成**（见 3.4，**GPT 出图优先**、即梦次之）——没实拍也能全 AI。
- **产品分级出镜**：产品**特写**只出现在"真实产品图垫底的图生视频"镜头里（保真有锚）；多场景 / 人物使用类镜头（文生视频）让产品以**中远景的"同类外形设备"**出现，并在提示词里写死"产品不给特写"——远景糊弄得过去，文生的特写必穿帮。验收时给了特写就重抽。
- 工具优先**即梦**，原理对可灵 / Runway / Sora / Veo 同样成立——它们都有图生视频 / 首帧能力，照搬即可。
- **即梦如有"多素材参考生视频"入口（Seedance 2.0 @引用）**：读 `references/seedance2-multimodal.md`——产品锚/场景锚/角色锚可用 @ 直喂替代 GPT 合成首帧，废片可编辑/延长不必重抽，BGM 卡点可前置到生成。真人脸素材会被拦截，注意。

### 3.2 六条铁律（带"为什么"，照做就不翻车）

**铁律 1 · 拆镜头，但别拆太多**
别让 AI 一条提示词连续生成十几秒——它会中途换场景、主体漂移。但也别堆一堆段（见第 2 步：≤15s 默认 2–3 段）。每个镜头只干一件事。

**铁律 2 · 统一设定分配到每条（重要的可用性约定）**
把"主体设定"（主角 + 产品的固定描述）和"风格后缀"（光线 / 背景 / 色调 / 画质）**直接揉进每一条分镜提示词里**，让每个代码块都能**整段复制**，用户不用再手动拼。

> 不要把统一设定单独列在最上面让用户自己往每条后面粘——那样很烦。直接分配好。
> 只在**确有差异**的镜头上做有意调整（如"痒点镜头不含产品""产品白底镜头去掉暖光后缀"），并标注这是有意为之，不是漏写。

**铁律 3 · 锁首帧（根治"开头乱加东西"）**
模型乱加效果，根源是**开头那一帧是它自己脑补的**。改用**图生视频 / 首帧**功能，上传一张干净、正常的参考图当起点，模型就没法在开头堆料了。然后提示词**只描述运动**。故事板模式下给每个节点都绑定参考图，等于把这招放大到全片。

**铁律 4 · 删掉"触发词"**
凡是描述"逐渐产生 / 大量堆积"的词，都会让模型从第 1 帧就糊上去。一律删除或替换：

| ❌ 触发词（让 AI 翻车） | ✅ 换成中性运动词 |
|------|------|
| 凌乱、打结、一团团、大量、堆积、喷涌、溢出、梳出毛发、脱落、密密麻麻 | 滑过、划过、整理、顺滑、贴合、缓缓移动、轻轻倒入 |

把"惨状 / 大量效果"隔离成**独立镜头**去渲染（且画面里不出现引发动作的工具），不要和"使用动作"塞进同一条提示词。

**铁律 5 · 效果用「静图」呈现，不靠连续生成累积（全 AI 路线，无需实拍）**
"一梳带出一团毛""倒出绵密泡沫""吸出黑头"这类**逐渐累积**的爽点是 AI 视频的死穴。但 AI 很会画**静态的"结果"**。所以把"过程"换成"结果 + 轻微运动"，三种做法任选，全程在即梦内完成：

- **方法 A · 首尾帧**：起点图（无效果）+ 终点图（**一小撮**效果），让即梦在两帧间补间，因果顺序天然正确。
- **方法 B · 结果图 + 图生视频**：先用即梦图片生成"带效果的静图"，再图生视频**只写运镜**，让它动一点点。结果靠静图保真，视频不负责"生成"。
- **方法 C · 蒙太奇硬切**：片段1（动作，干净）硬切 片段2（已有结果的特写），观众自行脑补"动作→结果"。最省事。
- **（可选）实拍 / 后期合成**：有条件时实拍那 1 秒最真，但**不是必需**。

> 关键：终点 / 结果图里效果只放"一小撮"，多了补间必糊。
> 同理：戏剧性的"使用前惨状"（满脸黑头、严重打结）也别让 AI 在连续视频里造——**做成一张 AI 静图**，再轻微运动，效果更可控也更真。

**铁律 6 · 产品本体镜头"光动机不动"（硬细节产品实测铁规）**
适用于电子设备、工业品、工具这类**买家天天摸真机、细节对不上就毁信任**的品类。图生视频里让产品**旋转超过约 20°**，或让**镜头移动**（上摇 / 跟随 / 环绕），模型就必须脑补它没见过的那一面——结果是结构被重新设计 + 凭空出现乱码 logo（实测：要求转 30° 它转到背面并把打印仓画成相机镜头）。对策：

- 提示词写死：**"外观、按键、屏幕内容严格保持与原图完全一致，不旋转，镜头固定不动"**，这句是防变形的关键，别删。
- 全画面**只允许一个运动元素**（光带扫过机身 / 屏幕渐亮 / 纸张吐出 / 激光扫描线），机身和机位全部锁死。"光动、机不动"照样有高级感，且零穿帮。
- 工具里有"运动幅度 / 速度"滑块就**拉到最低**；同一条提示词抽 2–4 次挑最不走形的（三抽一中是常态）。
- **特写不靠生成**：想要特写，后期在剪映里放大裁切，零成本且绝不穿帮。
- 用户没给某个面的图（侧面 / 背面），**就不要设计展示那个面的镜头**——AI 编不出来。
- **想要航拍 / 俯视 / 大场景镜头前，先问"产品这个角度有没有参考图"**（实测踩坑）：只有正面 / 斜视图时，纯俯视（80–90°）会逼 AI 编出"窝顶 / 机顶"这类没见过的面，logo 与刺绣必乱必变形。**要"大赛航拍气势"又要保真，就降到约 45° 高位斜俯（体育转播主机位）**——居高临下有气势，又仍能看见产品正面（门 / 窗 / logo），正好落在正面参考图锁得住的角度。纯俯视只能用在"产品当远景小目标、不展示 logo"的镜头。且这类出图务必**上传真实产品图当锚、写死产品形态约束，别纯文生图**。
- 验收标准：生成结果与原图**并排对比**按键位置、开孔、logo、顶部形状，对不上的直接弃，宁可画面朴素也不露馅。

> ⚠️ **品类边界（别套错）**：铁律6 只管硬细节品类。**美妆 / 香水 / 护肤 / 液体 / 食品饮料类反过来**——hero 镜头就该有水花 / 金色粒子 / 液体流动 / 倾倒挂壁等多运动元素互动（这正是高级感与"水润 / 质感"的来源），别套"光动机不动"把它拍呆。这类买家不逐部件比对，看的是氛围与质感。

> 同场补充：AI 生成的**小字必是乱码**（快递面单、标签、屏幕文字）。这类镜头成片里控制在 ≤2.5 秒、不定格，快节奏下无人细看；但**封面绝不用这些帧**。

### 3.3 配套技巧

**切点：⚠️ 本用户偏好无缝衔接（默认优先无缝，硬切仅快剪确需时用）**
通用带货逻辑是"节奏切卡BGM硬切"，但**本用户审美上排斥生硬硬切、偏好有设计感的无缝衔接**，所以为本用户出方案时默认反过来：
- **优先无缝**：尾帧承接（上镜尾帧导出当下镜首帧，真无缝）/ 遮挡切（有东西划过镜头瞬间切）/ 白闪+whoosh 掩护（强反差点）。剧情/叙事连续段尤其用无缝。
- **硬切**：只在快剪节奏确实需要、且无缝做不出来时才用，并说明理由。
- 黑场（藏变身那种）属功能性切，不受此偏好限制。

**转场必须带后备方案**：凡是**依赖生成内容才成立的转场**（如"标签纸吐出占满画面做遮挡切""杯子举到镜头前挡黑"），生成结果经常不给你那个画面（实测：纸只吐出半张就停了，遮挡不成立）。设计时就写明后备切法（通常是白闪或硬切），素材验收时哪个成立用哪个，不要为了转场去反复重抽素材。

**其他**
- **首尾帧锁因果**：首帧=无效果起点，尾帧=出现"一小撮"效果，模型补间，顺序就对。
- **首尾帧锁姿态（大幅动作的保底方案）**：转头 / 抬手 / 起身这类大幅动作，文字提示词常被即梦"缩水"到很小幅度（详见 3.5 ① 幅度条）。保底做法：**上传首帧图做参考，让图像模型出一张"动作已完成"的尾帧**（同场景同机位同光线，仅姿态变化，提示词注明"构图、景别、光线与参考图完全相同"），用首尾帧模式补间——幅度想多大画多大，不可能不够。运动提示词只写"平滑自然地完成该动作 + 颈肩自然联动 + 约 N 秒完成"。两个加分点：①尾帧可顺势**停在指向下一镜的姿态**（如视线看向下一镜主体的方向），衔接更顺；②补间若出现面部 / 肢体扭曲，把尾帧姿态减弱一档（如"全侧脸"改为"四分之三侧脸"）再试。
- **运镜方向全程一致**（都推近，或都平移），否则拼起来会顿。

**逐镜衔接规格（每个切点都写清，避免拼接跳 / 即梦跑偏）**
每两镜之间附一条 🔗，包含：①本镜尾帧定格在什么画面 ②下一镜首帧从什么画面起 ③切法（硬切 / 无缝）④切点落在哪一帧（动作顶点 / 遮挡 / 闭眼瞬间）⑤运镜是否延续 ⑥色调过渡（一致，或有意反差靠硬切完成）⑦卡 BGM 哪个节拍。

**空间保真（场景也要锚，三条规矩——实测翻车后固化）**
产品有真图锚，场景常常什么锚都没有：多条文生镜头各写"米色沙发浅色木地板"，每次生成 AI 都重新发明一个沙发——底缝高度、沙发脚、地板材质、光向、左右方位全是随机的（实测：同一沙发两镜底缝高度不同、木地板变瓷砖、玩具掉落点和营救点对不上）。文字锁不住几何，对场景和对产品是同一条定律：

> **场景先过"合理性"再谈保真（实测漏判）**：场景必须符合主体的生活/身份逻辑——家养宠物 → 居家/后院/公园（远处有房子、篱笆等居住痕迹），不是野生森林/丛林；室内产品 → 室内。再美再大的背景，只要和主体身份矛盾（猫狗跑在荒野丛林前）就出戏，还常顺手把本该有的居住感/本地化线索（如巴西彩色民居）换掉。出图后先问一句"这个主体真的会出现在这个环境里吗"。
1. **开拍前定"空间地图"**：一句话即可（如"沙发屏幕左、猫窝右后角、窗光从右、玩具掉在沙发中段底缝"），之后所有提示词的左右朝向、光向、物品位置按它写死，不许每镜即兴。**物品在哪儿丢的就必须在哪儿找回**——观众对"地点同一性"极敏感。地图里还要给**每个角色定"家位"**（睡点/基地/事发点分开放）——角色间的空间距离就是"奔赴""赶到"这些戏的来源，主角睡在事发点旁边，全片的位移戏都不成立（实测翻车）。
   **反命题同样成立："此处为空"也是已建立的事实**——某镜头的画面里某区域没有猫/没有人，后续镜头就不能让主体"早就待在"那块区域（实测：POV 镜建立了沙发前无猫，下一镜猫却睡在同一块地板上=凭空出现）。锚定参考帧时锚的是**地点**，不是"把所有镜头都锚到同一帧"——不同地点的镜头用各自的锚或按空间地图文生。
2. **同一地点第二次出现，用第一次的帧当锚**：从已过关镜头导出干净帧 → 图像模型把新主体合成进去 → 图生视频。沙发高度、地板、光向全部继承，物理上无法穿帮。
3. **叙事节拍防瞬移**：剧情类脚本按"信号 → 反应 → 行动 → 结果"逐拍点名，主体每次空间位移都要有对应的移动镜头（或明确的时间省略）；缺"反应拍"观众接不上因果，缺"位移拍"观众感知瞬移（实测：变身后直接出现在沙发边，观感极不适）。
4. **动线审计（剧情类必做，提示词是审计通过后的产物）**：骨架确认后、生成参考图之前，先出**动线表**——每镜一行：主体｜地点｜入画位置｜朝向与动作方向｜出画边缘｜情绪标签｜与下一镜的承接。动作方向尽量**一路同向**；有意的方向反转（如"反应朝声源、行动朝基地"）要在表里标注"有意"；**情绪标签连读应成弧线**（如温馨→试探→压迫→安抚→升华），情绪跳档和空间跳跃一样产生突兀感。网格/首帧图回来后，**先逐格核对实际朝向 vs 动线表**（对不上：无文字无产品的图水平镜像翻转即可改向，否则重抽），全对上才写视频提示词。
5. **可视化故事板（标注版）**：**剧情/演绎类默认在"骨架确认"环节就直接产出它——这不是可选升级项，是这类片的正确默认**（文字概要表锁不住动线，必翻车；用故事板对齐才是在正确的抽象层确认）。在动线表之上产出"带标注的故事板图"供双方共审——每镜=画面+标注+信息栏。标注用统一图例：红箭头=人物动向、蓝箭头=镜头动向、绿框=构图/空间焦点、橙箭头=光线方向、紫标=情绪张力；信息栏含：景别/运镜/焦点/光线/情绪/台词与声音/画面说明——**台词和音效在故事板层就逐镜入位**，声画一起设计。
   **标注图可直接当参考图指挥运镜（实测）**：在参考图上画红线标镜头路径，提示词写"严格按红线运动 + 红线仅为路径草图、成片绝不出现任何红线箭头标记"，模型照走位但不渲染线——这是控制运镜最直观的方式（见 controlled-generation-framework 维度6）。若不需要红线指挥、只是审美审计，则用"干净母版→叠标注审→裁母版出首帧"避免漂移。

**⚠️ 衔接不能与画面自相矛盾（防"人物凭空消失"）**
- "尾帧=首帧承接 / 镜头微移连续"**只用于主体与空间都连续**的镜头。
- 一旦**在场角色变了**（这镜是猫、下镜变狗或多出主播），那就是**独立构图的硬切**——**别写成"镜头微移到一侧、承接同一空间"**：小道具/小场景里轻微移镜不可能把原主体移出画外，会穿帮（如"窝前的猫被挪没了"）。
- 独立硬切要点明：下一镜**重新构图、谁在场**；上一镜的角色若不在本镜，用**时间省略**（"打烊后""换机位"）解释，或让它**继续在场**，二选一。

### 3.4 参考图怎么准备（首帧图 / 尾帧 / 结果图）

锁首帧、首尾帧、结果图都要静态参考图，没实拍也能全用图像模型生成。
> **图准备好后"怎么动起来"（反应/沉思双模 · 反应时长下限 · 三层动作分级 · 文字只补 4 类）→ `references/image-to-video.md`**——image-driven 的核心方法，写图生视频镜头前去查。
> ⚠️ **凡写「参考图N＝…」必落到用户已有的具体文件，缺图/需裁剪当场说清**：出全量那一刻就讲清每张参考图对应用户上传/已生成的哪一张（按画面特征点名）、用户没有的就明说要补、需从别张裁剪+放大的就讲明——别只给抽象描述等用户追问（完整规约见 `references/working-notes.md` 规约8）。

**🔑 用哪个模型出图**
- **优先 GPT 出图**（ChatGPT 图像生成）：**指令遵循强、能把产品文字 / logo 写对、可上传真实产品图做"放进场景"的合成**，复现特定产品（含刺绣文字、配色细节）保真度最高。
- **即梦图片次之**：同生态，喂即梦图生视频可能动得更顺；中文提示顺手。
  ⚠️ **即梦 / NanoBanana 用自然中文描述，别堆 MJ/SD 语法**（`--ar`、`--v`、`masterpiece, best quality` 这类堆叠英文质量词无效甚至干扰）：画幅在开头写"9:16 竖版"、质量写"8K 超精细"即可。
- 做法：**上传真实产品电商图让模型合成到场景**，比纯文生更像实物；同一镜可 GPT / 即梦各出一张，喂图生视频各测，挑动得稳的。

**🔑 用真人照片做身份参考时（图生图防"原图美化版"，带货上传卖家/模特脸必读）**
上传真人照，AI 默认生成"原图美化版"——背景、衣服、道具、坐姿、构图全沿用，只是把脸修精致。三步根治：
1. **第一句写死**："只把上传照片作为人物身份参考、用于保留本人真实长相，不要保留原照片的背景、衣服、道具、坐姿、拍摄环境和原始构图，重新生成一张全新画面。"——保留人，不保留原图。
2. **身份锚拆成具体清单**（"像本人"太泛，AI 会自动网红化）："严格保留脸型、五官比例、眉眼结构、眼睛、鼻子、嘴唇、肤色、年龄感、**发际线**、发型基础、整体气质" + 负面"不要变陌生人、不要欧美化、不要网红化、不要过度美颜、不要 AI 假脸"。发际线这类细节最影响像不像。
3. **发型三变量跟随原图、不硬套模板**：①长度跟原图（短发保短、长发保长，只优化蓬松层次）；②刘海跟原图（有就保有、没有别强加）；③发色写真化（原图黑发别写"纯黑/死黑"会吃掉层次，写"深棕黑/黑茶色/深咖"保通透）。
> 参考图给的是"感觉"不是"照搬"：服装道具要二创、不要复制参考图原物。

**九宫格"一图九帧"法（多镜头同一人物出镜时的首帧批量方案）**
全片多个镜头需要**同一个人物**时，别逐张生成首帧——逐张必变脸。改让图像模型（即梦图片 / GPT / NanoBanana 都行）在**一张 9:16 图里生成严格 3×3 九宫格**，每格对应一个分镜画面——因为是一次生成，九格的人物、穿搭、肤色、光感天然一致。提示词要点：

- 写死"**同一人物、同一穿搭贯穿全部九格，任何一格不出现第二个人**"
- **一致性锚点**：给人物固定 1–2 个小配饰（小耳环、同款手机壳），穿搭用**纯色、无图案、无字母**——既减少漂移，又防 AI 在衣服上印乱码
- 每格写一行画面描述（钩子 / 开箱 / 展示 / 试用 / 细节 / 场景…按分镜排）
- ⚠️ **裁格后必须先超清放大再用**：1080×1920 裁成九份，每格只有约 360×640，直接喂图生视频画质不够——用即梦超清 / upscale 放大到 1K 以上
- 产品特写格照旧要**上传真实产品图做参考**——九宫格解决的是"人"的一致性，替代不了"真图垫底"的产品保真流程
- **⚠️ 九宫格 vs 单张多抽的权衡（实践修正）**：九宫格一次出 9 格，模型注意力分散→脸不够像/表情动作重复/手易崩/各格质量不均。所以分场景用：①**快速看氛围 / 锁视频多镜首帧的跨格一致性** → 用九宫格（一次生成天然统一）；②**要高质量静态成片（人像组图/轮播图/精品主图）** → 改用**单张生成 6–9 次再选图拼**，模型注意力集中在一张，脸/手/光/构图都更可控，本质更接近"先拍很多张再选片"。带人脸的精品图别图省事直接九宫格。
- **组图防雷同的"覆盖矩阵"菜单**（人像/产品组图、轮播图、N张写真通用）：一组图先按多个维度排出覆盖矩阵，每张落在不同组合上，明令"整组不许角度/站位/构图/姿势雷同，每张是同一次拍摄的不同镜头语言选片"。维度清单：
  - **景别**：超广角大场景 / 远景 / 中景 / 近景 / 特写
  - **角度**：俯拍 / 仰拍 / 侧拍 / 正面 / 斜前方 / 逆光 / 特殊视角（如水面下、肩后探出）
  - **表情/眼神**（人像）：平静淡笑→闭眼轻笑→被逗笑→害羞→若有所思；看镜头/看对方/看镜外/闭眼（微表情曲线见 `references/subject-and-realism.md`）
  - **动作/姿态**：每张一个明确不同的动作（开场/穿越/互动/特写/高潮/收束）
  - **场景关系**：主体与环境/道具/配角的不同关系（独处/互动/尺度对比）
  典型一组 8 张就该覆盖"开场建立→穿行→大场景→中近景关系→视觉奇观→逆光特写→上升/收束高潮→极简收尾"这种弧线，而非 8 张同机位换姿势。
- **格数按需，不必凑九**：只缺 3 个镜头就出 1×3 或 2×2——格越少每格分辨率越高，可省掉放大步骤。**不含产品的镜头组是网格法的最佳场景**（产品 logo 在小格里必乱码，没产品就没这个短板）
- **网格可以"挂进"已有素材的空间**：把已过关镜头的导出帧、已过关的人物/角色帧一起上传当参考，提示词写"场景与参考图1一致、角色与参考图2一致"——新格生成的就不是新世界，而是旧素材宇宙里的新瞬间（配合 3.3 空间保真规矩 2）
- **需要首尾帧的镜头，把尾帧也排进网格**：如"爪子扫出玩具"这类"物体出现"型动作（铁律 5 的死穴），在网格里同时画"动作前"和"动作后"两格，裁出来直接当首尾帧用——同一次生成，连贯性天然成立
- **每格的主体朝向逐格写死**（实测翻车）：网格提示词里只给"奔跑格"写了方向、忘了给"营救格"写朝向 → 生成出奔跑向左、营救朝右，动线打架。空间地图的朝向规矩对网格的**每一格**生效，一格都不能省。补救：无文字无产品的格子可以**水平镜像翻转**改方向，零成本；有文字/产品的格子只能重生成

**链路依赖预先写入（一次性出全量提示词时必做）**
全量交付的提示词是一条生产链，链上每一环的输入是上一环的**生成结果**，不是设计稿。两条规则预埋偏差容差，避免生成结果一偏就全链返工：
1. **多张首帧图按序生成时，从第 2 张起，提示词里直接写明"上传第 1 张已生成的帧作为场景参考，场景、机位、光线与其保持一致"**——写进提示词，不靠事后口头提醒用户。
2. **图生视频的运动提示词里，凡是由首帧图决定的主体细节（人物/动物的姿态、服饰、表情、位置），一律写"与首帧图保持一致"，不要复述写死**——首帧图才是事实源；提示词写"站姿"而生成的图是坐姿，两者打架必出问题，且首帧一偏全部运动提示词都要返工。只有**产品关键锚点**（logo、刺绣、标志性结构）才逐项点名写死，因为这些不允许有任何偏差。
生成结果与设计稿有偏差但**效果更好**时（如设计"昂首站立"、生成出"呆萌端坐"反而更好笑），直接采用生成版并同步更新后续链路的锚点描述，不要为了贴合设计稿重抽。

**要点**
- **首帧图**：产品 + 主体的干净起始画面（也可用产品现成电商图垫底）。
- **尾帧 / 结果图**：带"一小撮效果"的画面（如"梳齿上挂一撮毛""装满浓稠果昔的杯子"）。
- **保持一致**：多张图用**同一张产品图 + 同一张人物肖像**当参考，产品 / 人不变样，再喂进视频。
- **真实感**：GPT 图偏精修时加 `photorealistic, candid phone photo, natural light` 找 UGC 感。
- **顺序**：先把图调满意再生成视频——图错了视频一定错。

### 3.5 分镜提示词输出格式（要详细到即梦无法跑偏）

**提示词太简单 = 即梦自由发挥 = 跑偏。** 每条都要把下面要素写全，一个镜头一段、整段可复制；并在镜头后附「🔗 衔接规格」（见 3.3）。
> **提示词字数甜区：约 120–250 字/条**——太短＝模型自由发挥出 generic；太长＝模型忽略后半段指令。写满要素但别注水。

**⚠️ 提示词全程"对模型说话"，不对用户说话（实测踩坑，所有提示词通用）**：提示词是给模型的**目标画面描述**，不是给用户的说明 / 叮嘱 / 谈判。两条铁规——
> ① **删掉一切元话术**：模型读不懂"擅自 / 关键 / 让人一眼看出 / 不许加速 / 我特意"这类**说给人听的话**（实测写过"颜色绝不擅自改成米白"——"擅自"纯是对用户说的，模型无视）。要约束就写**画面事实**（"底面与袜身同为黑色"），不写意图旁白。
> ② **正向描述里别点名"不想要的"颜色 / 物体**：在画面正向描述里写"不是米白""不要红色"，等于把那个词喂进模型、**反而诱导它生成**（图片生成模型尤其明显）。正面把**想要的**写死即可。负向点名只放进视频模型专门的**「避免」段**（如即梦图生的避免列表）——这与下文"避免段写禁止项"不矛盾：**区别在位置**（正向画面描述 vs 专门负向段）**和模型**（图片模型对正向里的负向点名更敏感，慎用）。

**① 提示词必含要素（按此顺序写）**
> 口诀（先记框架再抠细节）：**主体 ＋ 动作 ＋ 场景 ＋ 光影 ＋ 运镜 ＋ 风格 ＋ 画质 ＋ 约束词**。下面把每一项写到即梦跑不偏的颗粒度——口诀管"不漏项"，细节管"不翻车"。
- 景别 + 机位/角度 + 焦段视角（如"中景、固定机位略仰拍、约35mm"）
- 主体：**完整重述产品外形**（颜色 / 材质 / 每个标志性细节，**不可省略，否则即梦会改产品**）+ 宠物/人物（品种、毛色、姿态、表情）
- 动作：精确到轨迹、幅度、起止、速度，且必须符合物理逻辑（**一个镜头只做一个动作**）：
  - **主体朝向**：动物 / 人物头部朝向必须先定（如"犬头朝左、尾朝右"）——朝向决定了毛发生长方向，进而决定梳理 / 动作方向是否合理，不写朝向则后续所有方向约束都可能失效
  - **方向**：写起点和终点的画面位置（如"从画面左侧颈部划到右侧臀部"），不能只写"从左到右"；涉及顺序的动作必须符合物理逻辑——梳毛顺毛方向（颈→尾）、倒液体向下、推拉方向与受力一致
  - **速度**：必须给数值（如"每秒移动约 6–7cm""约 3 秒完成一次划过"），不能只写"缓慢"或"匀速"——没有数值 AI 会自己决定节奏
  - **持续性（防"动作中断/角色发呆"）**：连续动作类镜头（打斗、追逐、持续操作），AI 在一段里只能撑住 1 个动作节拍——一条 5s 提示词塞 3–4 拍（相撞→打飞A→打飞B→终结技），它演完一两拍就让角色静止发呆。对策：①**一镜一拍**，多拍动作拆成多段（每段 4s），**尾帧承接**串联（导出上段最后一帧当下段首帧），观感上是一场不间断的连续戏；②每段提示词写明"全程持续运动、无静止帧"并按秒排满动作；③"避免"里点名"动作中途停止、角色静止发呆、打斗暂停"；④总时长不够就**上调平台档位**（15s→21s/34s）换叙事完整，别硬塞。
  - **幅度（防"动作缩水"）**：即梦对大幅肢体动作（转头 / 抬手 / 起身 / 弯腰）默认只给 10–20% 的幅度——写"转头看向 X"实际只会动眼珠或微偏头。四个对策一起上：①**用终点画面描述幅度**，写"转到几乎只剩侧脸对镜头、鼻尖指向画面右下"而不是只写角度（AI 对终点状态比对度数敏感）；②**度数故意写过量**，要 45 度就写 70 度；③把大动作**提到提示词最前面**作为"核心动作（本镜唯一重点）"，并改用**固定机位**——运镜会分走运动预算，模型"动了镜头就少动人"；④在"避免"里**点名偷懒动法**（如"避免：只动眼珠不转头、转头幅度过小"）。文字提示词抽 2–3 次仍不够 → 直接上**首尾帧锁姿态**（见 3.3），保底必成
  - **物理反馈**：被作用的物体要有符合物理的响应，如顺毛梳理时毛发顺向平伏、出现光泽；逆毛则蓬起——不写这一层 AI 会让毛发纹丝不动，画面失真
  - **情绪前置 + 动词落地（人物情绪镜头）**：情绪词（"眉头紧皱、满脸烦躁"）放在**人物描述的最前面**——提示词越靠前权重越高，埋在中段会被忽略（实测：中段写"表情烦躁"，生成的人平静专注像好员工，钩子全废）。同时用**具体动作动词**落实情绪（用力拍打、重重摔下、叹气翻白眼）——AI 对动词的执行远比对情绪形容词可靠。精细小动作（牙咬纸、单指拨开关）做不稳，别写
  - **道具与产品区分**：痒点 / 惨状镜头里的"旧设备 / 竞品"道具，必须在**颜色和形态上与主推产品明确拉开**（产品是黑色就写"白色老旧、贴着胶带"）——否则观众会把吐槽对象误认成你卖的东西，对比变自黑
  - **主体真实感 & 品类工艺（有真人 / 穿戴 / 垂类 / 双人镜头时必读）→ `references/subject-and-realism.md`**：真人三限定（皮肤·重量感·眼神）、抓拍感表情、微表情逐秒曲线、穿戴面料剪裁词、配套装备写全、垂类行话锁专业、个人形象照扩展、人设方向、双人关系感——只在这类镜头才用到，写之前去查
- 运镜：方向 + 速度 + 幅度（**写小**，如"极缓慢匀速推近约10%、无晃动"）
- 光线：方向 + 色温 + 质感 + 阴影
- 背景：具体陈设 + 虚化程度
- 构图：主体位置 + 占画面比例 + 给字幕留白
- 风格/画质：电影感、真实质感、4K、色调
- 时长 + 节奏
- **产品形态约束（单独写，不能合并进"避免"一笔带过）**：
  - **光写"产品不变形"等于没写**——AI 不知道"变形"具体指哪里，必须把每个部件逐条列出：颜色、形状、数量、比例、朝向
  - 格式：先在主体描述里完整写一遍产品各部件，再在约束段里逐条镜像：
    ```
    产品形态约束（全程不变）：
    - 手柄：[颜色][材质][形状][特征细节，如防滑纹理位置、挂孔]
    - 支架：[颜色][形状，如 Y 形][连接部件，如螺丝数量位置]
    - 功能头：[颜色][结构][数量，如梳齿 10 齿][排列，如双排弧形]
    - 整体比例：[高宽比][朝向，如手柄朝下梳头朝上，不倒置不侧翻]
    避免 = 梳齿数量增减、支架形状改变、手柄颜色偏移、产品倒置、任何部件弯曲变形
    ```
  - 即梦在运动过程中会逐帧漂移产品外形，**必须在每一条提示词里都带产品形态约束**，不能只在第一镜写一次
  - **有使用过程的产品（挤出 / 开盖 / 涂抹 / 倒出类）还要加"行为约束"**：①开合方向写死（如"翻盖从正面向背面翻开，盖子保持连接，不是旋开"）；②内容物的颜色与质地写死，**并点名禁止项**（如"膏体为米杏色乳霜质地，绝不允许白色、透明、啫喱状、泡沫状"——不写禁止项，AI 默认挤出白色膏体）；③使用时的物理反馈写死（涂抹后的搓泥颗粒、按压后的回弹、倒出后的流动速度）。静态外形约束管"长什么样"，行为约束管"用起来对不对"，缺一个都会被懂行买家看穿
  - **⚠️ 别为"突出卖点"擅自给产品改色 / 改形（实测踩坑：把黑袜的毛圈底改成米白来突出减震）**：想强调某个功能时，本能会给它一个**实物没有的对比色**让它"跳出来"——这是**改了产品、违反图实一致**，买家收到货对不上就退货差评。**颜色 / 外形严格按实物**（黑袜的毛圈底就是黑色同色），卖点靠**纹理、厚度、动作（按压回弹）、光带扫过、特写打光**来体现，**不靠改色**。功能的"可见性"用镜头和光解决，不靠篡改产品本体。
  - **⚠️ 多配色产品：hero 选"浅色高对比"的那款，深色款留作系列展示（实测踩坑：黑款生成出来工艺全糊）**：产品有多个配色时，要展示**加厚 / 纹理 / 结构线 / 浮雕 / logo**这些卖点细节，就别用**黑色或深色款**当主推——黑色会"吃掉"细节，AI 生成出来一团黑、加厚和工艺看不见，正好埋掉你要卖的点。**hero 款（上脚 + 产品展示镜）选浅色或高对比配色**（米色 / 浅灰 / 白配亮色），加厚浮雕和 logo 一眼可读；黑色 / 深色款放进**结尾系列展示**当一员即可。选 hero 色还要兼顾和痛点旧物的区分度（痛点用纯白旧袜，hero 就别用纯白，用米色 / 灰拉开）。一句话：**显工艺看浅色，越深越糊。**
- **避免（负向约束）**：明确写"避免 = 产品颜色/造型改变、**新增或复制主体（多出第二只猫/狗/人）**、肢体畸形、数量或位置变化、镜头角度大幅漂移、画面扭曲、多余文字…"。⚠️ **图生视频在运镜幅度大时最容易凭空多出一只动物**——运镜尽量小，并**正向写明"画面始终只有一只 X、保持同一角度"**。
- **无文字尾缀（每条提示词默认带）**：句末加"**画面中无任何文字、字幕、标题、logo、水印、平台 UI**"——AI 生成的文字必是乱码，所有字幕 / 贴纸 / 标题一律后期在剪映里加，生成素材必须是"干净画面"。
- **人物镜头约束词尾缀（有真人 / 角色时默认带）**：句末加"五官清晰、面部稳定不扭曲、人体结构正常比例自然、同一角色服装发型一致"——泛用抽卡保底；要高保真仍按 `references/subject-and-realism.md` 写具体（具体 > 泛泛，同产品形态约束的道理）。
- **⛔ 无缝衔接约束（分镜路线每条都带，本用户硬规则）**：提示词和🔗衔接规格里**主动约束模型做无缝**（运镜方向全程一致／首尾帧承接／切点落遮挡瞬间／突变用遮挡或黑场掩护）——四种做法见 3.3 切点，此处不重复。每个切点写明用哪种无缝方式接，不写"硬切"（快剪确需的极少数除外且说明理由）。

**② 防跑偏四件套**：运镜幅度小且匀速 ｜ 一镜一动作 ｜ 单段时长尽量短 ｜ 每条带"避免"。每镜**多生成 3–5 条挑稳的**。

**③ 输出结构（每个镜头默认出「即梦」+「Veo」两套，都可复制）**：
```
[镜头N · 作用 · 时长]
—— 🅙 即梦版 ——
🖼️ 首帧图（即梦图片生成 · 注明上传哪张参考图锁产品）
<代码块：静态画面提示词 = 9:16 + 景别构图 + 产品完整外形 + 宠物/人姿态表情 + 场景 + 光线 + 写实风格 + 4K + 避免…>
🎬 图生视频（用上面首帧图）
<代码块：按①要素顺序的运动提示词（关键词式中文），整段写满>
—— 🆅 Veo 版 ——
<代码块：一段电影化自然语言（英文优先）写同一镜头：人物/产品外形 + 动作 + 运镜 + 光线 + 9:16；真人镜头把台词用目标市场语言直接内嵌（Veo 原生出声+口型）；句末加 clean, no on-screen text, consistent product>
🔗 衔接 → 镜头N+1：按 3.3 七项写清（两套通用）
⚠️ 本镜易翻车点：<一行写明本镜最可能崩的地方 + 验收时盯什么>
```
**关键**：即梦版首帧图必须是「可直接复制的图片生成提示词」，不能只写"参考某图"；Veo 版不堆关键词、用流畅电影化描述、台词内嵌。
**每镜的"⚠️ 易翻车点"必须具体到本镜**（如"logo 文字最易糊，出图后先确认可读再生成视频""泥条颜色必须灰白可见，太浅=爽点不成立""快速动作时盯手指是否崩坏"），不要写泛泛的"注意一致性"——这一行就是用户验收该镜的核对单。
**修订必须重发全量（一切交付物通用，不止分镜提示词）**：任何一镜的提示词改了（哪怕只改一两镜），都**重新输出完整排序的全套最终版**（含未改动的镜头，标注哪几镜是新版）——用户只看最新一条消息就能从头执行，绝不让用户跨消息拼"哪条是最新的"，看错版本比多打几行字贵得多。**这条对一切交付物生效**：剪辑教程 / 点击级手册、文案、执行表、脚本、故事板提示词——只要内容有任何变更（哪怕只改几个时间码、几句字幕、一个配色），都**重发完整版**，绝不只发"改动的那几条"让用户自己回上一条拼步骤（实测踩坑：只发改动的时间码，用户得跨两条消息对着剪，极易出错；本用户明确要求"变更必出全部教程"）。

### 3.6 Veo 版写法要点（和即梦的区别）
- **文风**：电影化**自然语言**（英文效果最好），一段连贯描述，别堆关键词。
- **🔑 首帧控制（Veo 同样要！别只发文字）**：产品 / 宠物镜头**默认用即梦生成的同款首帧图做 Veo 图生视频（image-to-video）锁产品**——纯文字 Veo 复现不了那个特定产品。英文描述只是叠在首帧上的"运动 + 台词"层。两套**共用同一批首帧图**。
- **台词与口型的取舍**：Veo 能原生出声 + 口型，但"图生视频是否带原生对白"看入口 / 版本：
  - 产品 / 宠物镜头 → 图生锁形，无台词，配 BGM。
  - 真人口播镜头 → ①图生(首帧=人+产品)锁形优先；②若该版图生不支持对白口型，改文生 + 参考图(ingredients)出原生对白；③或图生无声 + 字幕/外部配音兜底。
- **负向**：无独立 negative 框，把"避免"融进正向描述（clean, no text, consistent product, no extra animals）。
- **⚠️ 海外模型审核会拦"疼痛/受伤/冲击"措辞（实测：Veo 拦了痛点镜）**：痛点/惨状镜头描述身体痛苦的英文措辞——`lands HARD`、`knees buckling`、`wincing in pain`、`real pain`、`spent`、`hurt`、`injured`——会被 Veo / Sora 等海外视频模型的安全过滤**误判成"伤害/暴力内容"拦下**，哪怕只是个袜子广告。对策：把"疼痛/受伤"**改写成情绪向措辞**（累 tired、烦 annoyed、不适 uncomfortable、嫌弃 unimpressed、frowning / shaking his head），去掉冲击和受伤暗示，痛点钩子照样成立（"脚累 + 嫌弃表情"配后面爽点对比足够）。还被拦就再退一档：去掉揉脚等身体接触描写、纯靠表情叙事。**即梦等国产模型一般不卡这类运动疼痛表情**——同一痛点镜，即梦版照写、Veo 版用情绪向改写，两套分别适配。
- **时长**：单段约 8s，按它切；分镜 / 衔接 / 卖点逻辑与即梦**完全通用**，只是换了提示词文风。

### 3.7 口播驱动型视频（SRT 声画同步）→ `references/voiceover-srt.md`

当用户给口播文案 / SRT / 配音音频，或要做"边说边演示"对口型视频时读它（分段规则、音频模块格式、与铁律兼容、情绪三拍递进）。默认"BGM + 剪映后期配音"工作流不需要这套。
> **数字人 / 虚拟主播带货（合成人怎么不假、防恐怖谷）→ `references/ai-avatar.md`**：做数字人口播 / 数字代言时读它（两个出口 / 眼睛锚点 / 靠瑕疵显活 / 风格谱系）。

## 第 4 步：本地化文案

**面向投放的文案必须用目标市场的语言写，且是地道表达，不是逐字翻译。**

产出以下几块，都用目标市场语言（并附中文回译方便用户核对）：
- **钩子 / 开头字幕**：对应第 1 步的痒点，一句话抓住人
- **核心卖点字幕**：主打利益点，口语、短
- **具体数字 > 泛泛承诺（卖点 / 证言通用）**：能给数字就别给形容词——"3 天见效""省一半电费""从 12K 到 31K 用了 11 周"远胜"效果好 / 超划算"。具体数字 + 时间 + 可命名结果＝可信度，泛泛的"很好用"＝广告噪音。⚠️ 数字必须真实可兑现（虚标＝虚假宣传）。
- **CTA**：促单（如"手慢无""正在打折""点下方链接"）。**B2B / 工具设备类产品例外**：成交靠私信谈批量而非挂车冲动下单，CTA 改为"批量采购，私信拿报价"+ 发布后自己置顶一条评论（"需要报价的扣'报价'或私信"）+ 提醒用户开企业号并配私信关键词自动回复（否则非工作时间询盘流失）。B2B 泛行业打法还可在脚本里加一段**多行业场景快剪**（每场景 2–3 秒、产品只中远景），让不同行业的买家都能认出自己，时长放宽到 25–30 秒
- **话题标签**：混合「大流量泛标签 + 精准品类标签 + 本地化标签」
- **BGM 提示**：建议去目标市场区的 TikTok"飙升音乐"里选当地正火的，别用别地区的

**本地化注意**：语言要对（巴西是巴葡不是西语；魁北克是法语）；表达习惯、emoji、梗要贴本地；避免直译腔。

**人物形象本地化（重要）**：分镜里出现真人时，**必须指定目标市场的本地外貌**（族裔 / 肤色 / 发色 / 外貌风格）——即梦及多数模型**默认生成东亚脸**，不写就与市场不符、掉代入感。例：巴西用"moreno 小麦色肤色、深色头发、拉丁外貌"，并可按受众选白人 / 黑人巴西人。

## 第 5 步：合规红线（轻量提醒）

收尾给用户 2–4 条相关红线即可，不展开：
- **别宣称医疗 / 绝对化功效**（"治疗""根治""100%"易限流下架）
- **金融 / 虚拟币敏感词（双重雷区）**：理财·投资·回报·暴富·躺赚·传销·加密货币·比特币·Crypto——写进**文案/字幕**易平台限流封号（改"省下 / 划算 / 回本快"等中性说法）；写进**提示词**部分视频模型还会降级甚至拒绝生成（非必要别写）。
- **措辞合规（女性 / 室内 / 多人镜头防模型拒绝，实测反直觉）**：触发审核的常是**提示词措辞毒性**、不是画面本身。隐藏雷：夜店氛围 BGM 词（爵士钢琴 / Jazz Piano / Lounge / Bossa Nova / Smooth Jazz）+ 女性 + 室内 + 暖光＝被拦；删 会所 / 公关 / 包间 / 介绍美女；别把 4 个夜店元素锁一起（暖橙吊灯 + 背光屏 + 深色大理石 + 红木）。多人镜头**锁静态全画面**、别写"镜头摇到第 X 人"（触发换脸）；去掉人物-物体物理交互（别举杯 / 操作物）。安全替换＝纯物理词（120fps / HDR）+ 中性角色（"主理人"非"公关"）。与 §3.6 疼痛词、上条金融词同属"措辞致模型拒绝"家族。
- **尺度由用户和平台规则决定，不由助手主观替用户审查**：性感/暴露的尺度各平台不同、各目标市场不同、且随时变——不要凭"我以为平台会封"去主动删改用户内容。深V/收腰/显身材/泳装/内衣/cosplay 性感角色都是相关品类的正常卖点，默认保留。只在**用户明确要求**做合规过滤、或内容触及无争议的硬底线（如未成年、真实色情）时才介入；拿不准就**问用户的目标平台和尺度**，按用户的标准执行，别自作主张当审查员。
- **要风格、避 IP（防侵权的精确写法）**：想要某种风格但别碰具体版权角色/品牌——提示词写"完全原创角色、受 X 风格启发,但不模仿任何受版权保护的具体角色/形象、避免相似"(如"受柔和怀旧动漫启发但不模仿吉卜力具体角色")。**画风本身合法,具体角色/品牌形象才侵权**。同源手法:蹭场景用"虚构替代真实 IP"(瞎编队名台标,见画风轨道·伪直播截图)。
- **AI 内容按平台要求标注**
- **图文与实物一致**（尺寸、材质、效果别夸大，否则高退货 / 投诉）
- **按目标市场本地化合规**（如日本药机法对美容 / 医疗器械宣传管制严格；欧美对功效宣称也严）
- 涉及平台店铺（如 TikTok Shop）开通 / 资质，提示用户自行确认当地最新政策，别下断言

## 素材回看验收（用户发回生成片段 / 成片时）

用户把生成结果发回来时，**逐条抽帧看**（首 / 中 / 尾帧，视频文件可用 ffmpeg 抽帧），别只扫一眼缩略图。按三层检查：

1. **产品保真**：与原图并排对细节（按键、开孔、logo、形状），见铁律 6 验收标准。
2. **戏对不对**：画面质量高 ≠ 镜头成立。情绪镜头要看情绪真演出来没有、冲突感够不够（实测：画面真实但"演员"平静干活，钩子照样废，必须重抽）；场景镜头要看产品是否守住了"只中远景"。
3. **转场承接点是否成立**：设计好的遮挡 / 承接画面可能没生成出来 → 启用后备切法（见 3.3），别为转场重抽素材。

**常见可救情况（不用重抽）**：细节 / 图标在后段游移变形 → 只截取前段用；动作早早完成、后段静止 → 裁掉静止尾巴；首帧黑屏 / 渐入 → 保留，正好做亮度衔接。

**成片终检清单**（用户交回成片时逐项过）：
- 分辨率 **1080P**——剪映导出默认可能停在 720P，必须手动改（抖音对高清有加权）
- 字幕距底部 ≥15%、不压脸不压产品（平台底部 UI 会盖住贴底字幕）
- 字幕断行不拆词，断在标点处；放不下就缩字号或删字
- 配音与画面对齐；响度约 -16～-14 LUFS 即健康（剪映正常导出一般达标，可不强求测量）

## 后期剪辑（必须输出「剪辑执行表」，不是建议清单）

把 AI 生成的几段片段 + 参考图结果拼成成片。**默认交付一份逐条可照做的「剪辑执行表」**——用户打开剪映就能从上到下执行，不需要自己再决定"放哪、用什么、几秒"。包含以下四张表：

**⚠️ 精确卡点时间码必须基于"实际成片"，不能只按提示词设计值（实测踩坑，本用户硬要求）**：切点 / 字幕 / BGM 炸点 / 音效的**精确时间码，必须等成片生成出来、用 ffmpeg 抽帧读出真实节奏后再定**——AI 演绎 / 生成的实际节奏由模型自定，和脚本里的设计时间码**必然偏移**，只按提示词出的卡点必不准（本用户已多次发现）。所以默认工作流是**成片优先**：① 用户发回成片 → ② 抽帧读出每个节拍（痛点结束、产品登场、起跳、落地、收口…）的**真实时间码** → ③ 才出精确卡点版剪辑表。只有提示词、还没成片时**可以先给一版"设计值版"应急，但必须在最上面醒目标注**"⚠️ 时间码为脚本设计值，拿到成片后我按抽帧实际节奏重对一遍"，绝不让用户拿着错时间码去硬卡 BGM。抽帧不只用于保真验收（见上节），也用于读节奏定卡点——同一份抽帧两用。

> ⚠️ **卡点表内部不能自相矛盾：变速/补帧/裁剪在前，卡点在后（实测踩坑：给了"原始成片绝对秒数"又让用户把某段变速 0.85x，秒数当场失效，用户当场发现）**：别把"原始成片绝对秒数"和"之后还要把某段变速/裁剪"混进同一张表——变速会重定整条时间轴，那些绝对秒数立刻不准。正确顺序：**先做完全片补帧/变速/裁剪 → 再按变速后的最终时间线读秒定卡点**。更稳的是把音效/字幕**锚到"段＋视觉事件"**（"对到她睁眼那一帧"），绝对秒数只当参照，这样无论怎么变速都不脱拍。另：**单条整片演绎是一条连续片，格间转场即梦已渲染好，别再"切 N 段加转场"**——多此一举、徒增脱拍风险；只在要裁切口/叠 CTA 真图时才动刀。

> **用户剪辑零基础时，升级为"点击级手册"（实测用户反馈极好）**：每一步写到具体的菜单和按钮（如"顶部点【文本】→【默认文本】拖到轨道 → 右侧【朗读】标签 → 选音色 → 开始朗读"），并在最前面加两件事：①**素材重命名表**——让用户把 AI 生成的长文件名改成 1.mp4、2.mp4…按镜头编号，否则时间线必乱；②**粗剪保留段表**——每条素材保留哪几秒（如"镜头4：从头保留 2.5s，纸停住就切"），用户不用自己判断取舍。配音用剪映"文本朗读"（AI 配音，用户不用出声），文本块保留即字幕。判断依据：用户说过自己不会剪 / 第一次剪，或对话里问"哪个按钮"。③**时间码必须用剪映原生「秒:帧」格式给（实测踩坑：给"6.1s"十进制，用户在剪映看到的是 `6:03`、对不上）**——剪映按帧显示、不是十进制秒，默认 30 帧/秒（1 帧≈33ms）；开拍前让用户把项目帧率设 30fps，再按"帧=小数×30"换算（0.1s=3帧、0.5s=15帧、0.9s=27帧…）逐条给 `秒:帧`，并提醒"对不准就把播放头对到画面事件那一帧、秒:帧 只作参照"。绝不丢十进制秒让零基础用户自己换。

> **成片"显业余/生硬"的三大根因 + 修复（客户实测反馈，必修）**：用户反馈"场景间切换生硬、像业余"时，根因几乎不是转场特效，而是这三条——
> 1. **全片色调不统一**（每段 AI 片段各自色温，切一次跳一次="生硬"最大来源）→ 套**一个滤镜 + 一个调节层覆盖全片**，把所有片段压进同一色调（这是"全片统一色调"，区别于本节末「补充」里的"局部反差补色"——冷段压暗降饱和、暖段提饱和）。
> 2. **切口没卡在节拍上** → 用剪映**"自动踩点"**生成节拍点，把每个切口拖到最近的节拍点对齐（差半拍人眼就觉得顿）。
> 3. **切在静止帧上**（AI 片段动作做完就停，切在停顿处=幻灯片翻页感）→ **切在动作进行中**（纸还在吐、人还在动时就切走），带着动势进下一镜。
> 另：**叠化=PPT 廉价感**，删掉——但本用户偏好无缝，优先换成尾帧承接/遮挡切，而非生硬硬切（快剪确需时才硬切）。这几条做完，"AI 素材拼接"才变成"有人剪过的片子"。

**① 切点转场表（每个切点一行，全部写死）**
| 切点 | 时间码 | 转场类型 | 落点 | 配套音效 |
- 转场类型：无缝优先（尾帧承接/遮挡切/白闪+whoosh；偏好与做法见 3.3 切点），硬切仅快剪确需时用并说明理由；默认**不用**剪映花哨转场（旋转/翻页/溶解/叠化——一用就像广告，划走率飙升）。
> **备选转场库（非默认·知道有·特定情形可破例——偏好不变，好方法先存着）**：①**CTA 硬切**——有数据称 CTA 用硬切（而非软淡入）点击率更高，强转化/快剪结尾可破例硬切到 CTA 卡；②**叠化 ≤0.5s**——跨场景大跳的柔和过渡（虽非默认、偶尔比生硬硬切顺）；③**wipe 方向语义**——左→右=进步、下→上=成长、对角=动感（即便少用，方向是带意义的）；④**硬切卡 BPM**——快剪能量片每刀切在鼓点上=能量最大化。默认仍走无缝（见 3.3），这些只是工具箱备选。
- 落点 = 卡 BGM 哪一拍（如"phonk 骤停拍""副歌第 2 重拍"），精确到秒。
- 前 2 秒最关键：钩子字幕 + 强画面同时上；整体偏快剪。
> **6 种剪辑节奏公式（按内容选，非默认逐个套）**：①呼吸 [2s][2s][2s][6s] 循环（vlog/产品演示）②心跳 匀速→[0.5s]×5 加速（悬念/追逐）③波浪 [2s][3s][4s][5s]+静默（开箱/品牌/情绪弧）④子弹时间 [3s常速][4s 120fps慢][0.5s加速]（产品揭示/爽点顶点）⑤脉冲 [0.5s]×3[0.3s]×4[0.2s]×5[2s定]（预告/快剪促销）⑥静默重锤 [5–8s静][0.3s击][8–10s静][0.3s击]（高端/极简，配 luxury-aesthetic）。配套：**生成 2–3 倍素材库**（AI 不能补拍）、拼接时**裁掉每段 AI 片段首尾 0.5–1s**（首尾最不稳）。

**② 音乐 cue 表（BGM + SFX 逐点列出）**
| 音轨 | 入点–出点 | 内容 | 音量 |
- BGM：写明**去目标市场区飙升榜找什么关键词/曲风**（别用别地区的），以及结构（如"0–8s phonk → 8s 骤停 → 8.5–15s 八音盒萌曲"）。
- SFX 逐个列：时间码 + 音效名（whoosh / pop / 玻璃碎 / 唱片刮擦…）+ 作用。画面拍不了的物理事件（碎杯、坠落）**用画外音效补足想象**。
- **音频不对称（前后对比 / 痛点-爽点带货必用）**：痛点段和爽点段用**相反质感的音效**——痛点侧塞烦躁音（杂乱通知音、错误提示音、刺耳摩擦、卡顿声），爽点侧换干净音（清脆"叮"、顺滑 whoosh、满足落点音）。声音反差叠加画面反差，爽点"爽"得更狠，比只靠切 BGM 强。
- 音量：有口播 BGM 压 -12～-18dB；无口播可放满；SFX 比 BGM 高 3–6dB。
- **先关 AI 片段原声**：AI 视频工具生成的片段常自带各不相同的配乐/音效，拼接后互相打架。生成时能关音效就关；剪映里点视频轨道头的**喇叭图标一键关闭整轨原声**，只保留自己加的配音 / SFX / BGM 三层。这条要写进执行表第一行。

**③ 字幕逐条表（每条字幕一行，含精确位置）**
| # | 起–止时间码 | 字幕原文（目标市场语言） | 位置 | 样式/高亮 | 动效 |
- **位置写死**：默认"水平居中、距底部 18%（安全区内）"；产品/主体在下半屏的镜头改"距顶部 25%"——逐镜头判断字幕会不会压脸/压产品，写明每条放哪。
- **字体**：无衬线粗体（Montserrat Bold / Poppins SemiBold，按语言选）；主句占屏宽约 80%，每屏一句最多 2 行；白字 + 2–3px 黑描边。
- **高亮词逐条标出**：哪个词、什么颜色（优先用产品主色）、是否放大 1.3–1.5 倍。
- **动效**只用：逐句弹入 / 淡入 / 跟节拍逐词弹出，全片统一；有口播跟口播，无口播字幕即叙事。

**④ 导出参数**
- 9:16，1080×1920，≥30fps，mp4 / H.264；时长卡平台档位（TikTok 常见 15s / 21s / 34s）；按平台要求标注 AI 内容。
- ⚠️ **剪映导出分辨率默认可能是 720P，必须提醒用户手动改 1080P**——平台对高清有流量加权，720P 白白吃亏（实测踩坑）。

**⑤ 封面**
- **默认走"大字高对比"**：主体怼脸清晰帧 + 后期加大字标题（点击率逻辑：封面在信息流里是缩略图，大字 > 美学）。封面帧避开 AI 乱码小字的画面。
- **高审美封面 / 海报 / 详情图（海报流配方 · 时尚编辑四版面 · "任意主题当品牌"KV 系统）→ `references/cover-and-poster.md`**：高端产品图 / 系列片名帧 / 活动主视觉 / 系列海报才用到，做这类平面物料时去查。

**工具**：剪映 / CapCut 最顺手（自动字幕、卡点、模板）；也可 Premiere / 达芬奇。
**补充 · AI 人物动作"总想拉 0.8 倍"是通病（即梦/Seedance 实测，根因＋解法固化）**：AI 视频对人物动作常**略"赶"**——缺 ease-in/ease-out 与重量感、把动作压进太短时长、有效帧不足，叠起来就是"差口气、想拉慢"的不流畅感（人眼对生物运动极敏感，一点不自然就察觉）。两端解：
- **后期（最快）**：降到 **0.7–0.8 倍**，但⚠️**绝不裸变速**——直接拉慢会抽帧重复、反而更顿；**必须配光流补帧**（剪映：选片段→变速→开【智能补帧】选"光流法"；或先补帧到 60fps，达芬奇 Optical Flow / Topaz Video AI 同理）。"0.8x ＋ 光流补帧"才丝滑，裸变速是卡。慢镜美妆/奢品类降速还顺带做出"从容慢质感"＝加分。
- **生成端（治本）**：给足时长别让它赶（一镜一动作、动作多的格多给秒数）＋提示词点名"deliberate / slow cinematic motion / 约 N 秒完成"＋避开原地自转（最易糊＋顿）。
- 黑帮步等节奏片降速 0.7–0.8 配 phonk 同理；前后段色调反差不够就后期调色补（冷段压暗降饱和、暖段提饱和）。变速/补帧/调色都写进执行表。

## 发布前 · 观众反应压力测试（可选，脚本定稿后或成片后做）

钩子强不强、会被怎么喷，不要靠自我感觉——**模拟一群真实刷手的反应**来测。完整产出三件：

1. **3 秒划走判断**：完全代入睡前刷 For You 的普通用户（没义务喜欢这条视频，大部分 1 秒划走）——开头 1–3 秒会停还是会划？哪一帧 / 哪句话最可能留住手指？哪里最可能直接滑走？结论用来回炉改钩子。
2. **混合情绪评论区模拟（15–25 条，必含负面）**：按真实配比混合——真心种草 ~25%、问价 / 哪买 / 物流 ~20%、**质疑割韭菜 ~20%**（"looks cheap""temu 几块钱同款"）、玩梗跑题 ~15%、@朋友 ~10%、已购反馈 ~10%。全是好评的评论区一看就是假的，模拟也一样。**最大用途：把模拟出的高频问题和质疑，提前写成置顶评论和私信自动回复话术**——观众真问到时答案已经在那了。
3. **路人一句话总结**：这条像广告还是像真实分享？最可能被吐槽的点？给 1 条提升真实感 / 转化的修改建议。

进阶用法：①压力测试——把质疑比例拉到 50% 专找弱点；②多市场对比——同一条片按不同目标市场（含当地语言和网络用语）各跑一遍，看哪个钩子在哪个市场反应最好，出海选市场前低成本预演。
