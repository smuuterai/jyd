# 模式 B · 参考视频拆解 + 仿写

用户给了一条参考/竞品视频，想"基于原视频画面 + 声音做二次拆解"，并得到一个能换成自己产品的**仿写模板**时，按本文件执行。最终结果要能直接拿去做 TikTok UGC 仿写。

## 流程总览
1. 先用脚本对视频做**预处理**（抽帧 / 网格 / 抽音频 / 转写）。
2. 基于"画面 + 声音 + 产品出现 + 镜头节奏"做**七段拆解**（见下）。
3. 把第七段「仿写模板」交给**模式 A**（skill.md 主流程），套用户的产品出整套分镜与文案。

---

## 第 1 步：预处理（运行脚本）

```
python scripts/video_deconstruct.py <视频路径> -o <输出目录>
```
（Windows 若 python 不在 PATH，用 Python 安装目录下的 python.exe 全路径调用。）

脚本会在输出目录生成：`info.json`（时长/分辨率/帧率/音轨）、`frames/`（每秒 1 帧）、`scenes/`（转场帧）、`grid.jpg`（网格图）、`audio.wav`、`audio_analysis.json`（人声/BGM/静音粗判）、`transcript.txt`（口播转写或"不可用"说明）、`report.md`。

依赖 ffmpeg；转写需可选的 faster-whisper（没有就按"不确定"处理）。

**补充命令（脚本没覆盖时直接用 ffmpeg）**
- **关键片段高帧率抽帧**：每秒 1 帧看不清快动作。用户点名细看某段（或你判断某个关键动作需要逐帧理解）时，对该区间抽密帧：
  ```
  ffmpeg -ss 5 -to 10 -i 视频 -vf "fps=4,scale=480:-1" -q:v 2 片段_5s-10s/frame_%03d.jpg -y
  ```
  （默认每秒 4 帧、宽 480px；`-ss` 放 `-i` 前定位更快）抽完按每行 5 帧拼成长图通读动作全程。
- **转场帧数量不对**：场景检测阈值默认 0.25——转场帧太少降到 0.2，太多升到 0.3 重跑。

## 第 2 步：读取产物来分析
- **看画面**：用视觉能力读 `grid.jpg` 通览节奏，再按需细看 `frames/` 逐帧、`scenes/` 转场帧。
- **看声音**：读 `transcript.txt`（口播）+ `audio_analysis.json`（是否有 BGM/人声/停顿）。
- **定位产品出现秒数**：在帧里找产品第一次露出、每次露出的时间点。

### "不确定"铁律（必须遵守）
- 转写不可用或听不清 → **根据声音特征 + 画面做近似判断，并显式写"（不确定）"**。
- 看不清是字幕还是口播、看不清产品型号 → 同样标"（不确定）"，不要硬编。
- 不要泛泛而谈；**每个时间段都要具体到画面动作**（谁、做了什么、产品怎么动）。

## 第 3 步：按这个格式输出（七段，顺序固定）

### 一、核心判断
判断这个视频**真正让人停下来的原因**，不要只看字幕。分析：0 秒钩子、人物动作、产品露出、痛点呈现、使用过程、结果展示、CTA —— 指出其中哪 1–2 个是"留人"的关键。

### 二、时间轴拆解表
| 时间段 | 原视频画面/声音 | 停下来的原因 | 痛点呈现/痛点画面 | 产品出现秒数 | 卖点顺序 | 可复用结构 |
|--------|----------------|-------------|------------------|-------------|---------|-----------|
| 0–2s | ... | ... | ... | ... | ① ... | ... |

每一行都要具体到画面动作；产品出现秒数要标实际时间点。

### 三、镜头语言拆解
逐段分析：景别 / 镜头运动 / 人物动作 / 产品动作 / 转场方式 / 画面节奏 / 情绪变化。

**给每个分镜标价值等级**（仿写模板只继承高价值镜头，低价值直接删——仿写应比原片更紧凑）：

| 维度 | 高价值 | 中价值 | 低价值 |
|------|--------|--------|--------|
| 场景变化 | 明显切换 | 渐变过渡 | 无变化 |
| 动作变化 | 核心动作 | 辅助动作 | 重复动作 |
| 情绪变化 | 强烈对比 | 微妙变化 | 无变化 |
| 故事价值 | 推进关键 | 补充信息 | 可删除 |

### 四、口播 / 字幕逻辑拆解
拆成：Hook ｜ 痛点 ｜ 产品解决方案 ｜ 效果证明 ｜ 信任背书 ｜ CTA。
（口播不可识别处标"不确定"，并写出你据画面/语气的近似推断。）

**台词输出三版**（有口播的视频必须给全）：
1. **原版还原**：尽量还原原视频的用语、语气、停顿节奏；非中文保留原文 + 附中文释义。
2. **可直接复刻版**：不改表达逻辑，调成用户可直接拿去录 / 配音的版本（去掉听不清的部分、补顺语句）。
3. **转化优化版**：保留原风格，强化成交感——痛点更扎、卖点更具体、CTA 更明确。

### 五、产品卖点顺序
列出卖点出现的先后，并判断**为什么这样排**（如：先痛点再功效、先视觉冲击再讲参数）。

### 六、爆款结构总结
总结成一个**可复用公式**，例如：
```
0-2秒：冲突/痛点钩子
2-5秒：产品出现
5-10秒：使用证明
10-15秒：效果展示
15秒后：CTA
```

**附：奇观叙事弧**（"展示型卖点"产品可套——把产品效果拍成奇观）：
**亮出能力 → 能力爆发成奇观 → 奇观转为结果 → 收束归位**。如清洁剂=喷上→泡沫炸开铺满→污渍被分解→冲净如新；彩妆=一笔→色彩晕染成妆面→妆容成型→定妆光泽。本质是把"产品使用效果"放大成视觉奇观。
⚠️ **但原地高速旋转/万物爆发/复杂粒子特效是 Seedance 死穴**（花滑类旋转镜头实测糊）——奇观弧要落地，"爆发"那拍得控制在模型能扛的范围（铺开、晕染、生长 OK；高速旋转成球、万剑齐发大概率崩）。满屏高速特效的玄幻理想稿基本生成不出来，别被漂亮提示词骗了。
**快速判据（看到这类提示词先过一遍）**：真正必崩的是**原地自转/旋转 + 复杂粒子神兽特效 + 唱跳口型同步**这类。注意区分（冲浪案例修正）：**带方向的高速位移（奔跑/冲浪/跳跃/腾空转体）能做好，原地自转才是死穴**——别把所有"高速"都判死。同时出现"原地旋转 + 多重粒子特效 + 多主体对嘴"两项以上≈必崩纸面稿。当前模型甜区其实比想的宽：单主体或少主体、方向性连续运动（含高速）、单一/可控特效都能成；窄区是原地自转、满屏高速粒子、精准唱跳对嘴。**对外部炫酷提示词的默认态度：先问"它的难点是自转还是位移、特效是单一还是满屏"，再判能不能做——别凭"看着炫"就一刀切判死。**

**附：通用种草九段弧**（没有参考视频、从零起盘 C 端种草内容时可直接套，视频可合并相邻段）：
钩子（强情绪自拍/冲突）→ 开箱（俯拍、包装随意摊开）→ 展示（自然光手持看细节）→ 试用/上身 → 核心卖点特写 → 真实场景使用（半/全身）→ 用后满意反应 → 生活感摆放（产品融进日常杂物）→ CTA（手势引导，画面无文字）。
按品类选动作：穿戴类=试戴上身；工具家电类=使用演示；美妆食品类=涂抹/试吃；其他=安装设置过程。

### 七、仿写模板
把结构改写成**通用模板**，把具体产品/场景换成占位符（如 [产品]、[痛点]、[使用动作]、[效果]），方便替换成其他产品。模板要标明每段的：时长、画面要点、口播/字幕要点、产品出现时机。

### 八、产品替换检查单（用户要"换成我的产品"时必做）
逐项核对原视频的动作逻辑换了产品后还成不成立，不要直接套：
- **原产品怎么被使用**：拿法 / 戴法 / 摆法 / 开合 / 倒出 / 涂抹——新产品的尺寸、形态、重量支持同样的动作吗？不支持就改成新产品的自然用法（如原视频"单手转着展示戒指"，换成手持终端就改"双手递到镜头前 + 按键演示"）。
- **哪些镜头必须保留**：钩子镜头、节奏结构、产品首次露出的时间点——这些是爆款骨架，动了就不是仿写了。
- **哪些必须同步改**：与原产品功能绑定的台词 / 字幕 / 特写动作；新产品的核心卖点要顶替原卖点出现在同一个时间槽位。
- **新产品的最佳露出方式**：按新产品的形态重新选（小物件适合近手特写、设备适合操作演示、穿戴品适合上身效果）。
- 替换后回到模式 A 出分镜时，**产品特写仍必须走"真实产品图 + 图生视频"锁保真**（skill.md 铁律 3 / 6），不能靠文字描述新产品。

---

## 第 4 步：接回模式 A（出可生产的仿写）
拿第七段模板 + 用户的新产品，走 skill.md 主流程：
- 用模板的结构/节奏当脚本骨架
- 按"全 AI 路线"出每段**分镜提示词**（锁首帧 / 无触发词 / 效果用静图）
- 出目标市场**本地化文案 + 字幕 + BGM 建议**
- 给**后期剪辑**清单（见 skill.md 后期章节）

这样就完成了"拆解爆款 → 仿写成自己产品的可生产方案"的闭环。

## 可选交付物：离线拆解报告

默认在对话里交付拆解结果。仅当用户要"**可转发 / 可存档的完整报告**"时，额外产出：
- **HTML 自包含报告**：七段拆解 + 关键帧图整合成单个 HTML 文件，**所有图片 base64 内嵌**（`data:image/jpeg;base64,...`），离线可看、转发不丢图。
- **图片 ZIP 包**：网格总览图 + 转场帧 + 片段长图打包。
不要主动做——多数场景对话交付更快，报告只在用户明确要求时生成。
