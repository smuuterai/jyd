# TikTok Product Video Planner

> *把一张产品图和一个目标市场，变成能直接生产的短视频方案，而且先把 AI 视频最容易翻车的地方堵住。*

**这个 Skill 面向 TikTok / Reels / 抖音 / 视频号的带货短视频策划：痒点、脚本、即梦与 Veo 分镜提示词、本地化字幕、剪辑执行表、合规提醒，一次交付。**

[效果示例](#效果示例) · [快速开始](#快速开始) · [触发方式](#触发方式) · [它会交付什么](#它会交付什么) · [文件结构](#文件结构) · [验证](#验证)

---

## 它解决什么问题

你不是缺一个“帮我写脚本”的助手。你真正缺的是一个知道 AI 视频会在哪里坏掉的人：产品一动就变形、黑头第一帧就糊满鼻子、宠物多出第二只、字幕被平台 UI 盖住、剪映导出还停在 720P。

这个 Skill 把带货短视频拆成可执行的生产链：先判断产品和市场，再选生成路线，再写可复制的分镜提示词，最后给剪辑执行表。它的重点不是文案漂亮，而是让成片更可控。

---

## 它会交付什么

| 模块 | 交付物 |
|---|---|
| 痒点与卖点 | 1 个主冲突、1 个主利益点、一句话定位 |
| 视频脚本 | 默认 15 秒的时间轴表，标明镜头数量和素材来源 |
| AI 分镜提示词 | 即梦版 + Veo 版，含首帧图、图生视频、产品形态约束、易翻车点 |
| 本地化文案 | 目标市场语言的钩子字幕、卖点字幕、CTA、话题标签、BGM 建议 |
| 后期剪辑 | 切点转场表、音乐 cue 表、字幕逐条表、导出参数 |
| 参考视频拆解 | 抽帧/音频预处理脚本 + 七段拆解模板 + 仿写模板 |

## 效果示例

输入一句：

```text
我想在巴西 TikTok 上卖一款宠物开结梳（蓝灰色双排不锈钢刀片、浮动刀头、防滑手柄，约17.5cm），用即梦做一条15秒的视频，帮我出一套完整方案。
```

输出会像这个黄金样例：[Brazil TikTok Pet Grooming Brush](examples/pet-grooming-brush-brazil.md)。

它展示了完整交付形状：路线选择、痒点与卖点、15 秒脚本、即梦 / Veo 分镜提示词、巴葡字幕、剪辑执行表和合规提醒。

## 快速开始

本地使用时，确保目录结构里有根文件 `SKILL.md`：

```text
jyd/                # 仓库根目录
  SKILL.md
  README.md
  evals/evals.json
  references/
  scripts/
```

发布到 GitHub 后，一行安装命令：

```bash
npx skills add smuuterai/jyd
```

装完后对 Agent 说：

```text
我想在巴西 TikTok 上卖一款宠物开结梳，用即梦做一条 15 秒带货视频，帮我出完整方案。
```

- “剪映怎么剪，给我具体执行表”

## 它和普通提示词有什么不同

| 维度 | 普通临时提示词 | 本 Skill |
|---|---|---|
| 产品保真 | 常让模型自由发挥 | 真图垫底、产品形态逐条约束、硬细节产品执行“光动机不动” |
| 累积效果 | 直接写“梳出一堆毛”“吸出黑头” | 用首尾帧、结果静图或蒙太奇硬切呈现 |
| 生成路线 | 默认逐镜或随意选择 | 先判断整片演绎 vs 逐镜分镜，并说明推荐理由 |
| 本地化 | 翻译标题和标签 | 按目标市场语言、人物外貌、平台节奏和 CTA 写 |
| 剪辑 | 给建议 | 给剪映可照做的切点、音效、字幕、导出参数 |
| 参考视频 | 只总结 | 抽帧、转写、七段拆解、输出可替换产品的仿写模板 |

## 安全边界

- 不承诺“100% 爆款”或“保证转化”，只交付可执行的创意生产方案。
- 不擅自调用外部付费 API；参考视频拆解脚本只处理用户提供的本地视频。
- 不把平台政策当成固定事实；涉及 TikTok Shop、广告资质、医疗/美容功效宣称时提醒用户核对当地最新规则。
- 不鼓励使用真实品牌、影视角色、赛事队名或人物肖像做侵权擦边素材；需要风格时使用原创替代写法。
- 如果用户要拆解竞品视频，输出的是结构仿写模板，不复制原片的独特表达、商标或受保护素材。

## 验证

先跑静态体检：

```bash
python scripts/check_evals.py
```

再用 `evals/evals.json` 里的 3 条 prompt 做人工回归。第一条 prompt 的输出形状可对照 `examples/pet-grooming-brush-brazil.md`。合格输出至少要包含：

- 痒点/卖点、脚本结构、分镜提示词、本地化文案、剪辑执行表、合规提醒；
- 分镜提示词明确使用首帧/首尾帧/结果静图来处理累积型效果；
- 即梦版和 Veo 版都能整段复制；
- 目标市场文案使用目标市场语言，而不是直译腔；
- 修改任一镜头后重发完整排序版，避免用户跨消息拼版本。

参考视频拆解依赖 `ffmpeg` 和 `ffprobe`。没有它们时，`video_deconstruct.py` 会给出安装提示；`check_evals.py` 会把它标成 WARN，而不是静默通过。
