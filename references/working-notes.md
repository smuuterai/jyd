# 协作工作规约 + 资源索引（让技能"拷一个文件夹=全套"）

> 本文件把原先只存在"记忆"里、但对用本技能有用的两类东西固化进技能文件夹：①协作工作习惯（怎么跟用户配合）②外部资源指针。换电脑只拷技能文件夹时，这些不会丢。

## 一、协作工作规约（每次用本技能都遵守）

1. **先确认，再出全量（无例外）**：接到任何"出提示词/帮我生成"请求——视频、单张图、写真、海报、场景大片，**都先确认形态(图/视频/先图后视频)+数量(单张/一套)+用途+画幅**，再出简短骨架让用户点头，最后才展开全量。常犯错=把"出图/出单条"当例外闷头出分镜，没有例外。动手写分镜前自检一句："形态和用途我确认过了吗？"

2. **先选生成路线（弹选项+给推荐）**：出分镜前必用提问弹"A整片演绎 / B逐镜分镜"两个选项，标出推荐和理由，用户选定才继续——不替用户默默拍板，不惯性上逐镜。多主体/连贯剧情→推A；硬细节特写/要精确控制→推B。

3. **提示词写全写细，让模型跟控制走**：通盘列尽这个项目已知的全部失败点，一次写全（身份/空间/时间/突变/过程/机位/动作密度），不被动等用户逐条纠。**核心：不预测模型、只规定模型**——心里一冒出"它应该会…"就是个漏洞，立刻写成显式指令。想到的问题自己用提示词解决，不要用"提醒/要不要先…"把问题抛回用户。

4. **修订必重发全量**：任何一镜改了，重新输出完整排序的全套最终版（标哪几镜是新版），用户只看最新一条消息就能从头执行，绝不让用户跨消息拼版本。

5. **外部材料：有用的吸收、无用的舍弃**：用户发来提示词/教程/库时，先列"强项(值吸收)/弱项(不照搬)"清单给用户，确认后只合并验证过或补短板的，与实测冲突的舍弃，绝不整体照搬。**下结论标准要硬**：单条成片抽帧没崩=弱证据(换题材可能翻)，真实发布拿数据=强证据，别把弱的说成强的。
   - **判"可吸收/已覆盖"前先逐文件读完再下结论**，别凭 README/摘要抽样就断"没别的了"（实测吃过亏，半读全库就下错结论）。
   - **与用户偏好冲突 ≠ 删**：偏好决定"默认"、不决定"知识库边界"。有效但和偏好冲突的好方法，标成"非默认备选 + X 情形可破例用"保留；只有"实测更差 / 越界 / 侵权"才真不收。

6. **技能内容必须自包含、可移植**：写进技能的内容用泛化品类描述（"某工业设备项目""某萌宠项目"），不写私有项目代号/型号/自创剧情名；规律+为什么+怎么做要完整，可信度靠"实测确认/已验"撑。

7. **用户全局偏好**：①转场**优先无缝**（尾帧承接/遮挡切/白闪掩护），不喜欢生硬硬切，硬切仅快剪确需时用；②剪辑零基础——方案以 AI 生成为主，剪辑只留剪映点击级操作；③产品零重绘，用官方图/实拍图后期贴。

8. **参考图清单必须落到用户已有的具体文件，缺图/需裁剪当场说清（实测踩坑：出全量时写"参考图2＝产品罐图、参考图3＝绿贴特写"，既没对应到用户上传的哪一张、也没说图3 其实要从图2 裁，用户被迫追问才补全）**：任何提示词里出现「参考图N＝…」时，**在出全量那一刻就一次讲清**三件事——①它对应用户**已上传/已生成的哪一张具体文件**（按可识别的画面特征点名，别只给抽象描述让用户自己猜哪张）；②若某张参考图**用户手里根本没有**，明说"需要你补拍/补出一张 X"或"可从已有的 Y 裁剪+放大得到"；③可选的参考图标注"可选，缺了也不影响"。原则：**参考图清单是给用户照着备料用的，不是写给我自己看好看的**——"用哪张、缺哪张、裁哪张"必须主动说清，绝不等用户问。

## 二、外部资源索引（按需查，不批量吸收）

做具体品类的图/海报/人物图卡壳时，去这些案例库找灵感——**弹药库，不是方法论**；方法照本技能走，案例去这里查，别批量吸收（纯堆料）。

- **awesome-gpt-image-2**：github.com/freestylefly/awesome-gpt-image-2（在线 gpt-image2.canghe.ai）——506+ GPT Image 2 案例+模板，含产品电商/品牌Logo/海报/人物一致性卡/商业摄影。
- **opennana**：opennana.com/awesome-prompt-gallery——1万+ 案例，覆盖 Nano Banana / GPT Image / Seedance 2.0。
- **elementsix-skills / seedance-storyboard**：github.com/elementsix/elementsix-skills——Seedance 故事板模板+案例。⚠️方法与本技能 seedance2-multimodal.md 重复且本技能更深，只当模板参考库，不吸收方法。**（已逐文件深读确认：SKILL.md/quick-reference/10 示例全读，硬参数·@语法·故事板·运镜·救镜头·一镜到底全被我们更深覆盖，零方法增量。）**
- **fpv-immersive-video-prompting**：github.com/zhouwei713/fpv-immersive-video-prompting——FPV 沉浸视频提示词 skill，比本技能窄（纯 FPV、无带货层）。**（已逐文件深读确认：SKILL.md+5 references+2 examples 全读。~85% 与已吸收重复：红线路径控制·一镜巡游·排除格线编号·去标记全已更深覆盖。已收 2 真增量进 controlled-generation-framework §6：①红线↔编号停靠二选一决策；②非人类 POV 身体物理写死。其余[数量-时长表 / N 张资产包]＝弱启发或带货相关性低，不收。）**
- **luban**（已装为独立技能）：github.com/LearnPrompt/luban-skill——打磨/审计技能本身用，不是做视频用。

**处理原则**：用户贴来某条具体案例 → 按"有用的吸收、无用的舍弃"过筛（有方法增量才吸收，纯案例只当灵感）；图库整体不吸收，只记位置。
