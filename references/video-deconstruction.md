# 模式 B · 参考视频拆解 + 仿写

用户给了一条参考/竞品视频，想"基于原视频画面 + 声音做二次拆解"，并得到一个能换成自己产品的**仿写模板**时，按本文件执行。最终结果要能直接拿去做 TikTok UGC 仿写。

## 流程总览
1. 先用脚本对视频做**预处理**（抽帧 / 网格 / 抽音频 / 转写）。
2. 基于"画面 + 声音 + 产品出现 + 镜头节奏"做**七段拆解**（见下）。
3. 把第七段「仿写模板」交给**模式 A**（skill.md 主流程），套用户的产品出整套分镜与文案。

---

## 第 1 步：预处理（运行脚本）

```
python scripts/video_deconstruct.py <视频路径> -o <输出目录>
```
（Windows 若 python 不在 PATH，用 Python 安装目录下的 python.exe 全路径调用。）

脚本会在输出目录生成：`info.json`（时长/分辨率/帧率/音轨）、`frames/`（每秒 1 帧）、`scenes/`（转场帧）、`grid.jpg`（网格图）、`audio.wav`、`audio_analysis.json`（人声/BGM/静音粗判）、`transcript.txt`（口播转写或"不可用"说明）、`report.md`。

依赖 ffmpeg；转写需可选的 faster-whisper（没有就按"不确定"处理）。

**补充命令（脚本没覆盖时直接用 ffmpeg）**
- **关键片段高帧率抽帧**：每秒 1 帧看不清快动作。用户点名细看某段（或你判断某个关键动作需要逐帧理解）时，对该区间抽密帧：
  ```
  ffmpeg -ss 5 -to 10 -i 视频 -vf "fps=4,scale=480:-1" -q:v 2 片段_5s-10s/frame_%03d.jpg -y
  ```
  （默认每秒 4 帧、宽 480px；`-ss` 放 `-i` 前定位更快）抽完按每行 5 帧拼成长图通读动作全程。
- **转场帧数量不对**：场景检测阈值默认 0.25——转场帧太少降到 0.2，太多升到 0.3 重跑。

## 第 2 步：读取产物来分析
- **看画面**：用视觉能力读 `grid.jpg` 通览节奏，再按需细看 `frames/` 逐帧、`scenes/` 转场帧。
- **看声音**：读 `transcript.txt`（口播）+ `audio_analysis.json`（是否有 BGM/人声/停顿）。
- **定位产品出现秒数**：在帧里找产品第一次露出、每次露出的时间点。

### "不确定"铁律（必须遵守）
- 转写不可用或听不清 → **根据声音特征 + 画面做近似判断，并显式写"（不确定）"**。
- 看不清是字幕还是口播、看不清产品型号 → 同样标"（不确定）"，不要硬编。
- 不要泛泛而谈；**每个时间段都要具体到画面动作**（谁、做了什么、产品怎么动）。

## 第 3 步：按这个格式输出（七段，顺序固定）

### 一、核心判断
判断这个视频**真正让人停下来的原因**，不要只看字幕。分析：0 秒钩子、人物动作、产品露出、痛点呈现、使用过程、结果展示、CTA —— 指出其中哪 1–2 个是"留人"的关键。

### 二、时间轴拆解表
| 时间段 | 原视频画面/声音 | 停下来的原因 | 痛点呈现/痛点画面 | 产品出现秒数 | 卖点顺序 | 可复用结构 |
|--------|----------------|-------------|------------------|-------------|---------|-----------|
| 0–2s | ... | ... | ... | ... | ① ... | ... |

每一行都要具体到画面动作；产品出现秒数要标实际时间点。

### 三、镜头语言拆解
逐段分析：景别 / 镜头运动 / 人物动作 / 产品动作 / 转场方式 / 画面节奏 / 情绪变化。

**给每个分镜标价值等级**（仿写模板只继承高价值镜头，低价值直接删——仿写应比原片更紧凑）：

| 维度 | 高价值 | 中价值 | 低价值 |
|------|--------|--------|--------|
| 场景变化 | 明显切换 | 渐变过渡 | 无变化 |
| 动作变化 | 核心动作 | 辅助动作 | 重复动作 |
| 情绪变化 | 强烈对比 | 微妙变化 | 无变化 |
| 故事价值 | 推进关键 | 补充信息 | 可删除 |

### 四、口播 / 字幕逻辑拆解
拆成：Hook ｜ 痛点 ｜ 产品解决方案 ｜ 效果证明 ｜ 信任背书 ｜ CTA。
（口播不可识别处标"不确定"，并写出你据画面/语气的近似推断。）

**台词输出三版**（有口播的视频必须给全）：
1. **原版还原**：尽量还原原视频的用语、语气、停顿节奏；非中文保留原文 + 附中文释义。
2. **可直接复刻版**：不改表达逻辑，调成用户可直接拿去录 / 配音的版本（去掉听不清的部分、补顺语句）。
3. **转化优化版**：保留原风格，强化成交感——痛点更扎、卖点更具体、CTA 更明确。

### 五、产品卖点顺序
列出卖点出现的先后，并判断**为什么这样排**（如：先痛点再功效、先视觉冲击再讲参数）。

### 六、爆款结构总结
总结成一个**可复用公式**，例如：
```
0-2秒：冲突/痛点钩子
2-5秒：产品出现
5-10秒：使用证明
10-15秒：效果展示
15秒后：CTA
```

**附：奇观叙事弧**（"展示型卖点"产品可套——把产品效果拍成奇观）：
**亮出能力 → 能力爆发成奇观 → 奇观转为结果 → 收束归位**。如清洁剂=喷上→泡沫炸开铺满→污渍被分解→冲净如新；彩妆=一笔→色彩晕染成妆面→妆容成型→定妆光泽。本质是把"产品使用效果"放大成视觉奇观。
⚠️ **但原地高速旋转/万物爆发/复杂粒子特效是 Seedance 死穴**（花滑类旋转镜头实测糊）——奇观弧要落地，"爆发"那拍得控制在模型能扛的范围（铺开、晕染、生长 OK；高速旋转成球、万剑齐发大概率崩）。满屏高速特效的玄幻理想稿基本生成不出来，别被漂亮提示词骗了。
**快速判据（看到这类提示词先过一遍）**：真正必崩的是**原地自转/旋转 + 复杂粒子神兽特效 + 唱跳口型同步**这类。注意区分（冲浪案例修正）：**带方向的高速位移（奔跑/冲浪/跳跃/腾空转体）能做好，原地自转才是死穴**——别把所有"高速"都判死。同时出现"原地旋转 + 多重粒子特效 + 多主体对嘴"两项以上≈必崩纸面稿。当前模型甜区其实比想的宽：单主体或少主体、方向性连续运动（含高速）、单一/可控特效都能成；窄区是原地自转、满屏高速粒子、精准唱跳对嘴。**对外部炫酷提示词的默认态度：先问"它的难点是自转还是位移、特效是单一还是满屏"，再判能不能做——别凭"看着炫"就一刀切判死。**

**附：通用种草九段弧**（没有参考视频、从零起盘 C 端种草内容时可直接套，视频可合并相邻段）：
钩子（强情绪自拍/冲突）→ 开箱（俯拍、包装随意摊开）→ 展示（自然光手持看细节）→ 试用/上身 → 核心卖点特写 → 真实场景使用（半/全身）→ 用后满意反应 → 生活感摆放（产品融进日常杂物）→ CTA（手势引导，画面无文字）。
按品类选动作：穿戴类=试戴上身；工具家电类=使用演示；美妆食品类=涂抹/试吃；其他=安装设置过程。

### 七、仿写模板
把结构改写成**通用模板**，把具体产品/场景换成占位符（如 [产品]、[痛点]、[使用动作]、[效果]），方便替换成其他产品。模板要标明每段的：时长、画面要点、口播/字幕要点、产品出现时机。

## 🎭 梗结构库（创意弹药，活库——每拆一条真实爆款就往里填一条）

**为什么有这库**：梗/故事创意是本技能最弱环——空想不稳、静态库会过时。唯一靠谱来源是**拆真实爆款**（市场已验证有人买账）。本库是"活的"：用户发来觉得好的爆款 → 拆解 → **抽象成"可复用梗结构"填进下面**（不囤具体案例，囤结构）。想梗时先过这张库，套不上再空想。

**怎么往库里填（每条格式）**：
```
【梗结构名】（一个能复用的母题名，如"拟人职场""反差变身"）
- 内核：为什么戳人（一句话点破心理机制）
- 结构骨架：N 拍的抽象流程（去掉具体产品/角色，留占位符）
- 适配品类：哪类产品套得上
- 套用示例：换 1-2 个不同产品演示怎么套
- 来源：拆自哪条爆款（标平台+大致数据，证明market验证过）
- 易翻车点：用 AI 做这个梗时哪里会崩（接生成判据）
```

**已沉淀的梗结构**（起步先放几个通用母题，后续靠拆爆款扩充）：

1. **拟人/赋予身份**——把产品或动物当人/职业看待（"我家猫转正成 CLT 打工人""这台机器是车间劳模"）。内核：荒诞的身份错位 + 观众代入自己。适配：萌宠、家电、工具。易翻车：多主体/复杂动作按生成判据控。
2. **反差变身**——平凡状态 → 某契机 → 高能反转（猫钻窝变超人、素颜→上妆）。内核：期待落差的爽感。适配：美妆、服装、有"使用前后"的品。⚠️ 变身是状态突变，按 seedance 文件"藏突变/拆段"处理。
3. **痛点惨状 → 救场**——先演真实的烦/糗/无效努力，产品登场解决。内核：先共鸣痛、再给解药。适配：几乎全品类。配"无效努力"节拍（见 skill.md 第2步）。
4. **格式挪用**——把别的内容格式套在产品上（新闻快讯/监控实拍/豪宅看房/选秀/教父办公室）。内核：熟悉格式 + 错位主角 = 反差喜感。适配：任何想要高级感/新鲜感的题材。
5. **奇观化**——把产品效果拍成视觉奇观（见上"奇观叙事弧"）。
6. **反差萌 × 实力背书 × 魔性卡点**——精致活泼的人物，放进一个"反差环境"里跟着魔性神曲卡点蹦跳/对口型，产品自然出现在手上身上。内核：①精致女孩×硬核场景的视觉反差勾人；②魔性 BGM+夸张活泼=洗脑完播；③真实场景（工厂/仓库/档口）顺带做"源头/实力"信任背书，不用开口讲。结构骨架：[张臂灿笑登场，亮出反差环境] → [跟神曲卡点，在场景里不同位置蹦跳/比划/对口型，产品挎身上/戴上/举手里顺势露出] → [举产品比心+联系方式] → [账号/搜索框 CTA]。适配品类：服装/包/首饰/配饰/小商品等"挂身上就能展示"的品，尤其**源头工厂/档口卖家**（背景即背书）。来源：抖音工厂女孩 Call Me Maybe 卡点舞，约 8000 赞。易翻车：①魔性靠真人表演张力+对口型，AI 生成真人跳舞对口型不稳（属生成窄区），**这条更适合真人实拍、或半 AI**；②若全 AI，人物活泼表情和卡点动作是难点，按真人三限定+情绪前置写，动作别太复杂。
   ✅ **真人实拍版已验证可行**：用真人模特（本地化外貌，如巴西辣妹）+ 自己的真实仓库/货架（一墙产品=天然"源头货多"背书）+ 葡语卡点字幕，仿出成片效果好、带货属性比原片更强。优化点：带货镜头要给产品 0.5s 近景特写（别全程全身看不清货）、CTA 加"为什么现在买"的钩子、加一个"手划过整排货"强化款多。**有真人+真实档口/仓库的卖家，这条梗是低成本爆款路子，优先真人实拍而非 AI。**
7. **荒诞反差钩子 × 满足感结果奇观**（"可见去除物/前后对比"类产品的高完播带货结构，市场验证）。内核：①前 2 秒用**意外/荒诞的人设或主体组合**（如女仆装壮汉给羊驼梳毛）制造"这什么鬼？"三连问、拦截滑动——钩子不靠产品、靠画面荒诞；②全程用**可见的大量"去除物/结果"**（梳出一大坨毛、吸出黑头、擦掉污渍）做 ASMR 满足感＝完播引擎；③多对象轮换续新鲜。结构骨架：[反差人设/奇观主体 把产品怼镜头 + 卖关子台词"秘诀是 X 不是 Y"] → [对象1→2→3 轮换，每段"使用动作 + 可见大量结果"成对 + 一句痛点卖点] → [最大那坨结果奇观顶点 + 打消顾虑卖点] → [红箭头指向挂车 + 紧迫感 CTA，首尾呼应回钩子画面]。适配品类：**有"可见去除物/前后对比结果"的品**——宠物去毛梳、去黑头/去角质、清洁去渍、洗车、疏通、收纳整理。套用示例：篮球减震袜→"大坨毛"换成"落地震感前后对比"、钩子用"西装上班族突然球场暴扣"的反差、多球员/多场景轮换；去黑头仪→意外场景做钩子、"吸出物"特写做满足感。来源：拆自巴西 TikTok 宠物去毛梳带货（@Lucas Silva，约 6–7 万播放，女仆装男给羊驼/狗/杜宾梳毛、葡语字幕）。易翻车点：①荒诞反差靠真人表演张力 + 真实主体，全 AI 难复刻"女仆装+真羊驼"的荒诞真实感→**优先真人实拍/半 AI**；②"可见大量结果"是累积型效果、AI 视频死穴→走 AI 就用静图呈现结果（铁律 5）、别让演绎连续"梳出一大坨"；③多对象轮换若全 AI 要锁角色/产品一致性。

8. **长 ASMR 教育信任型**（"可见结果"类产品的长视频甜区，与 #7 短奇观配成长短一对）。内核：不靠拦截靠"养着看"——①单主体全程慢做的 **ASMR 解压感**当完播引擎（梳毛/清洁/去黑头的过程本身耐看）；②边做边**口播教育**（讲原理 + 强调"温和不极端"，如不剃秃/不拉扯/不伤），把十几块小件讲成"懂行的人在教你"＝信任；③点"主体很享受/不抗拒"打消顾虑；④情感 CTA。结构骨架：[软钩子＝当季/场景痛点一句 + 立刻进入解压动作] → [边做边讲原理 + 打消顾虑（温和/不极端）] → [可见大量结果顶点] → [主体享受反应 + 设计细节背书] → [挂车 + 情感收尾"别让你的 X 再受 Y"]。适配品类：同 #7 的去除型/可见结果品，尤其**客单稍高、需建立信任**或**做精准转化**时。时长甜区 60–90s。套用示例：宠物去毛梳→单猫慢梳 85s 讲"夏天去浮毛不剃秃"；去黑头仪→单人慢清+讲原理+"皮肤不疼"。来源：拆自西班牙 TikTok 宠物去毛梳带货（单猫 85s，西语口播教育）。易翻车点：①长视频靠口播撑，AI 真人长口播+口型不稳→真人实拍或口播配音+ASMR 画面更稳；②"主体享受"靠真实表情，AI 难演自然，优先真人/真宠。

（↑ 以上是起步母题 + 已拆爆款，**真正的价值靠后续拆真实爆款不断往里加新结构**，尤其本地化、时效性强的新梗。）

**用法**：做新品想梗时，先扫这张库，挑 1-2 个母题往产品上套；想要新鲜/本地化梗时，回到"模式 B 拆解"流程拆一条当下爆款，拆完把新梗结构补进本库。

## 🧩 去除型/可见结果产品·组件库（6 条同品类跨市场爆款拆出，换品直接选配）

**适用品类**：宠物去毛梳、去黑头/去角质、清洁去渍、洗车、疏通、收纳整理——一切"有可见去除物 / 前后对比结果"的品。

**核心定律**：这类爆款 ＝ **可换的钩子 × 可换的痛点 × 可换的 CTA × 不可换的"可见结果引擎"**。"梳出/吸出/擦掉一大坨"的可见结果是命根子、永远保留；钩子/痛点/CTA/时长是换品换市场时要重新选配的变量。下面三张菜单是 6 条爆款（巴西葡语 / 西班牙西语 / 美区英文 + 无字幕）拆出的已验证选项。

**① 钩子菜单（5 种已验证拦截法）**
- **荒诞人设奇观**：意外人物/装扮当主体（女仆装大汉给羊驼梳毛）——博眼球最强。
- **软教育/痛点直入**：一句当季或恐惧痛点立刻进入解压动作——门槛低、留对的人。
- **摆拍剧情反转**：前几秒高戏剧/高好奇场景（警察拦截），第 4 秒反转到产品——可借爆款剧情。
- **规模/极端奇观**：选**最极端的"前状态"**（巨型重度打结狗）+ 列队/巨物 scale 冲击——主体越惨、去除越爽。
- **开箱真实感**：拆快递抽出产品——UGC 真实感 + 暗示"有货能发"，门槛最低。

**② 痛点菜单（恐惧型 > 麻烦型）**
- **恐惧型（强）**：主体会受伤/健康出事（"猫舔食掉毛吞进胃/毛球"）。
- **麻烦型（中）**：脏 / 累 / 家里到处是 / 夏天热。
- 选型规律：能戳"主体受伤/健康"就别只说"麻烦"，恐惧转化更狠。

**③ CTA 杠杆菜单**
- 挂车指向（红箭头+橙车）｜情感收尾（"别让你的 X 再受 Y"）｜社会证明（"大家都在用"）｜全程促销横幅（always-on 紧迫感）｜**物流信任（"今日特惠+快速发货"，跨境带货必备）**。

**④ 两个放大器**
- **新奇主体加成**：越意外的主体（羊/兔/马 > 猫/狗）拦截越强。
- **无字幕跨市场**：不依赖口播＝任何语言市场都能投，靠 ASMR 画面 + trending sound 传播。

**⑤ 时长 × 结构**：短 15–25s ＝ 奇观拦截快剪（梗 #7）；长 60–90s ＝ 单主体 ASMR 教育信任（梗 #8）。同款产品两种都能成，按"博流量 / 做转化"二选一。

> 换品用法：留住"可见结果引擎"，从①②③各选一个钩子/痛点/CTA 配新品，再回模式 A 出分镜。来源：6 条同品类（宠物去毛梳）跨市场爆款拆解沉淀。

### 八、产品替换检查单（用户要"换成我的产品"时必做）
逐项核对原视频的动作逻辑换了产品后还成不成立，不要直接套：
- **原产品怎么被使用**：拿法 / 戴法 / 摆法 / 开合 / 倒出 / 涂抹——新产品的尺寸、形态、重量支持同样的动作吗？不支持就改成新产品的自然用法（如原视频"单手转着展示戒指"，换成手持终端就改"双手递到镜头前 + 按键演示"）。
- **哪些镜头必须保留**：钩子镜头、节奏结构、产品首次露出的时间点——这些是爆款骨架，动了就不是仿写了。
- **哪些必须同步改**：与原产品功能绑定的台词 / 字幕 / 特写动作；新产品的核心卖点要顶替原卖点出现在同一个时间槽位。
- **新产品的最佳露出方式**：按新产品的形态重新选（小物件适合近手特写、设备适合操作演示、穿戴品适合上身效果）。
- 替换后回到模式 A 出分镜时，**产品特写仍必须走"真实产品图 + 图生视频"锁保真**（skill.md 铁律 3 / 6），不能靠文字描述新产品。

## 第 4 步：接回模式 A（出可生产的仿写）
拿第七段模板 + 用户的新产品，走 skill.md 主流程：
- 用模板的结构/节奏当脚本骨架
- 按"全 AI 路线"出每段**分镜提示词**（锁首帧 / 无触发词 / 效果用静图）
- 出目标市场**本地化文案 + 字幕 + BGM 建议**
- 给**后期剪辑**清单（见 skill.md 后期章节）

这样就完成了"拆解爆款 → 仿写成自己产品的可生产方案"的闭环。

## 可选交付物：离线拆解报告

默认在对话里交付拆解结果。仅当用户要"**可转发 / 可存档的完整报告**"时，额外产出：
- **HTML 自包含报告**：七段拆解 + 关键帧图整合成单个 HTML 文件，**所有图片 base64 内嵌**（`data:image/jpeg;base64,...`），离线可看、转发不丢图。
- **图片 ZIP 包**：网格总览图 + 转场帧 + 片段长图打包。
不要主动做——多数场景对话交付更快，报告只在用户明确要求时生成。
