# Seedance 2.0 多模态参考生视频（即梦新模型能力档案）

来源：dexhunter/seedance2-skill（基于字节官方文档）。当即梦入口支持"多素材参考生视频"（@引用）时按本文件用；老版图生视频仍按 skill.md 主流程。**使用前先确认用户的即梦版本/会员档位有没有这个入口。**

## 输入约束（硬参数）

- 图片 ≤9 张（jpeg/png/webp/bmp/tiff/gif，单张 <30MB）
- 视频 ≤3 个（mp4/mov，单个 <50MB，总时长 2–15s）
- 音频 ≤3 个（mp3/wav，单个 <15MB，总时长 ≤15s）
- 总文件数 ≤12；生成时长 4–15s 可选；分辨率 480–720p
- ⚠️ **写实真人脸部素材（图片和视频）会被系统自动拦截**——真人口播工作流上传素材前先处理（AI 生成人脸可用，真实照片人脸不行）

## @引用语法（每个素材必须声明用途，只写"参考@图片1"=废话）

| 用途 | 写法 |
|---|---|
| 首帧/尾帧 | "@图片1作为首帧""@图片2作为尾帧" |
| 人物/角色锚 | "参考@图片1的人物形象" |
| 场景锚 | "场景参考@图片3" |
| 产品锚 | "产品细节参考@图片3" |
| 运镜复刻 | "参考@视频1的运镜效果" |
| 动作编排 | "参考@视频1的动作编排" |
| 节奏卡点 | "根据@音频1的节奏卡点""视频节奏参考@视频1" |
| 音色/BGM/音效 | "旁白音色参考@视频1""背景BGM参考@音频1" |
| 服装/字体 | "穿着@图片2的服装""字体参考@图片2的字体" |

组合示例："@图片1的人物作为主体，参考@视频1的运镜和动作编排，背景BGM参考@音频1，场景参考@图片2"

## 结构化分层模板（做系列内容/锁统一风格时用）

复杂或系列项目，提示词按分层写（而非一段散文），便于"风格固定、只换分镜"：
```
core_style: [年代/风格/色调/胶片质感…]        ← 系列固定，写一次
visual_quality: [画质/颗粒/色散/光晕…]         ← 系列固定
character_modeling: [每个角色的外形/服装/发型逐项] ← 系列固定（=角色锚）
script_and_storyboard: [logic + 逐 shot：景别+动作+台词] ← 每集只改这层
audio_and_post: [音效/BGM/配音风格]
```
做系列（如同一 IP 多集）时，前三层复用，每集换 storyboard，风格和角色天然统一。⚠️ 但 storyboard 层仍要按六维度补全（时间分配/衔接动作/藏突变），外部模板的分镜层普遍只写状态、不写过程和时间，照抄会犯"跳过程"的老坑。

**多格故事板的动作不重复纪律**：12/16 格这种长故事板，要求"每格动作明显不同、不能雷同"，否则演绎出来注水拖沓。设计时逐格检查动作差异度。

**⭐ 死穴判据修正（极限运动项目推翻旧结论，重要）**：之前说"高速复杂动作必糊"是错的，一刀切了。精确规律是——
- **真死穴 = 原地自转/旋转**（花滑转圈、舞蹈旋裙、产品360°旋转）：自转时模型要凭空生成不断变化的背面/侧面，必糊。
- **能做好 = 带方向的高速位移**（奔跑、冲浪、跳跃、飞驰、腾空转体）：朝一个方向的连续运动模型有迹可循，实测冲浪+空中抓板+慢动作完整不崩。
- 判断一个高难动作能不能做：问"它主要是**绕轴自转**还是**沿方向位移**"——自转避开或改慢，位移放心做。
- **绕开自转的实操**：题材本身含旋转（花滑/舞蹈）时，别写"原地高速旋转"，改成"**移动旋转 + 低位跟拍**"或多用滑行/伸展/急停等位移动作——成片会呈现带位移的转而非糊掉的自转。低机位贴地跟拍 + 怼脸特写收尾，是运动题材稳出片的组合。

## ⭐ 可复用模板：真人运动类一镜到底（多条极限运动/冰上项目验证，高度可复制）

三条同类成片提炼出的固定配方，换题材即可复用，**带货首选运动/户外/运动服饰/运动鞋品类**（核心卖点就是"动起来什么样"）：
1. **前置全规格角色卡**锁人物/服装/装备（见上"角色卡完整规格"）
2. **开场怼脸特写**：主角脸/上半身近景，头发飘动、表情专注无畏
3. **运镜起手式**：镜头从前方 → 流畅绕到身后 → **降到极低机位（贴地/贴板/贴水面）并锁定该高度直到结束、绝不升高**
4. **贴地低位后追**：前景是轮子/冰刀/浪/沥青纹理，强调速度与接地感
5. **全程位移类动作**（雕刻/滑行/加速/急停），不写原地自转
6. **高潮**：起跳腾空 + **切戏剧性慢动作** + 空中转身抓装备 + 直视镜头 + 服装/头发飘起（这一帧=封面/主视觉）
7. **落地**：镜头随冲击震动但锁在身后 → 立即恢复 → 继续冲 → 冲出画面收尾（一镜到底不停）

**带货翻译**：飘逸服装在动感中展示版型面料 ｜ 贴地后追拍鞋/护具/装备细节 ｜ 角色卡锁"品牌脸"做系列号 ｜ 腾空慢动作那帧做封面 ｜ 一镜到底=原生运动 vlog 感，无广告腔，适配信息流。

**运镜分阶段锁定写法**：镜头分段写并锁死——"先怼脸上半身 → 流畅绕到身后 → **降到冲浪板高度并锁定在该高度直到结束、绝不升高**"。"到达某位置后锁死+明令不许变"比泛泛"跟拍"精确得多。
**连续镜头内变速**：高潮瞬间（起跳/腾空）"保持同一镜头不剪，切戏剧性慢动作"——Seedance 能在一镜内执行变速强调，提升质感。

## 真人带货镜头的景别组合（顺口写法）

真人+产品同框的带货镜头，用一句话锁住"既看清产品又有人建信任"：**"产品中近景特写，人物中远景相结合，镜头平稳，动作连贯"**。配合 @图1锁人物、@图2锁产品。这是"产品分级出镜"（skill.md 3.1）的提示词落地短语。注意：这只是景别骨架，时间分配/衔接动作/真人三限定/负面词仍要按六维度框架补全——外部带货案例普遍只写到这层就完事，是它们粗糙之处，别照抄那个"简"。

## 与本技能锚定体系的映射（核心价值）

主流程的三类锚直接翻译成 @ 角色，**可替代"GPT 合成首帧"那道工序**（少一道合成、少一层漂移）：
- 产品锚（真图垫底）→ "产品细节参考@产品图"
- 场景锚（空间保真规矩 2 的旧帧）→ "场景参考@锚定帧"
- 角色锚（九宫格/B帧）→ "人物形象参考@角色帧"

两条路线并存：①GPT 合成首帧 → 图生视频（老路，可控性已验证）；②多素材 @ 直喂（新路，省工序）。**新项目可各跑一条 A/B，挑稳的**。

## 救镜头工具箱（废片不再只能重抽）

- 延长："将@视频1延长15秒"（向后）/"向前延长10s"（向前），分时段描述新增内容
- **Extend 接龙做长视频（超单段时长时用，Grok Imagine 实例，即梦/可灵的"延长"入口同理）**：一段生成 → "从上一段的**确切最后一帧无缝继续**，保持 100% 一致（人物/服装/场景/风格）" → 再接一段 → 再接。把 15s 滚成 27s、33s。每段开头必写"从确切最后一帧无缝继续 + 列出要锁的一致性项"，结尾写下一段的起始状态。带货用于：完整产品演示流程、稍长口播种草等单段装不下的内容。⚠️ 各工具的"延长/Extend"入口和效果不同，用前确认当前工具有没有这个功能。
- **"余韵"收尾设计（高审美品牌片提质）**：最后一段专门做优雅淡出——粒子/元素缓缓飘落消散、动态平息、音乐化作长混响淡出、柔光 bloom 或渐暗，留下情绪余韵。比"定格+CTA"高级，适合香水/珠宝/奢品的品牌片。写法："元素缓缓失去强度、优雅飘落、化作柔光消散；声音化作丰富长混响缓缓淡出；柔和电影感淡出，给出完整情绪收束，masterpiece 级、有余韵的结尾"。
- 角色替换："视频1中的女主唱换成图片1的男主唱，动作完全模仿原视频"
- 元素修改："将视频1女人发型变成红色长发"（逐个改视觉元素）
- 剧情改写："颠覆@视频1里的剧情"，保留画面大部分
- 动作没执行的废片（如"主体没完成该动作"）可先试"编辑@废片：让主体完成 X 动作"再决定是否重抽

## 分时段结构（8s 以上可用，谨慎）

```
0–3秒：[开场画面、运镜、动作]
3–6秒：[中段发展]
6–10秒：[高潮]
10–15秒：[收尾、定格]
```
⚠️ 与主文件动作密度铁律的裁决：**分时段 ≠ 可以塞拍**——只用于口播段（3.7）和一镜到底场景推进，每个时段内仍只排 1 个动作节拍。

## 它的禁忌表（与实测互证）

引用必须声明用途｜同段不写冲突指令（固定镜头 vs 环绕二选一）｜内容量配时长（4–5s 别塞多场景）｜每张上传图都要 @ 标用途｜音频指导必写（明显提升质量）｜不传真人脸

## 与铁律 6 的冲突裁决

它有"产品悬浮 360° 旋转"模板——多图参考下保真可能改善，但**未实测前**：硬细节品类（电子/工业/带 logo 文字）仍执行"光动机不动"；软体无字品类（毛绒/布艺）可以试旋转。实测出结果后回写本节。

## 故事板大图 → 漫改演绎（一图整段路线）

配套玩法：先用图像模型一步生成"专业故事板大图"，再整张 @ 进 Seedance："将@图片1以从左到右从上到下的顺序进行漫画演绎，分镜切换以及重点情节演绎加入特殊音效"。适合剧情连贯、单格保真要求不高的内容（人物剧情、萌宠故事）；**不适合**产品细节特写多的硬保真内容（单格分辨率低 + 格内编号污染，不能裁格当首帧）。它管"剧本层"，动线审计（skill.md 空间保真 4/5）照做。

**⛔ 整片一次出（演绎）必须有完整故事线束缚模型，否则必散（用户硬规则）**：选了"整片15秒/演绎"路线，绝不能丢给模型一堆松散画面让它自由发挥——**必须有一条清晰的故事线**（起因→发展→转折→结果，或明确的情绪/动作弧线）贯穿所有格，每格是这条故事线上的一个节拍。故事板大图本身就要画出这条线（格1到格N是连续剧情，不是N张独立美图）。无故事线的演绎=模型放飞=画面散、留不住人。**先有故事，再有故事板，再有视频。**

**⭐ 控制点在"图"不在"文字"（萌宠剧情项目实测确认）**：故事板演绎模式下，Seedance 是在**读 6 张格子图**来演，不是读你的文字——文字写得再细再碎也没用，反而误导。**所有细节必须画进故事板的格子里**（动作、产品、表情、构图全靠图说话），喂给 Seedance 的文字只需简短点明"按格演绎+一致性锚+负面词"。这与逐镜路线**完全相反**（逐镜靠文字写到极细）。用错控制方式（对着演绎模式猛写文字）会产生"完全失控、全靠它自己跑"的错觉——其实是该用图控却用了文字控。
**实测确认成立**：带刺绣 logo 的毛绒产品整段运动中**全程清晰不糊**，6 格全演到、不丢格、主角不变脸、竖屏正确——故事板演绎路线对带货可用。

**⭐ AI 文字判据修正（多人宣传片项目实测）**：旧铁律"AI 文字必乱码、一律后期加"要精化——**大号、标准、全大写的短英文单词（品牌名/国名/年份，如 SALE / BRAZIL / 2026 这类）现在的模型基本能写对**；**仍乱的是**：小字、长句、副标题、中文、手写体（实测一条多人宣传片里大标题正确、副标题仍乱码）。所以：标题级大写英文短词可以让 AI 生成并验收，其余文字仍后期加。产品 logo 刺绣属"图形+短词"，实测也能保真。

**⭐ 特效判据修正（多人宣传片项目实测，重要）**：旧说"复杂特效快闪必崩"太宽。精确规律——**崩的是"主体本身的高速自转/形变"；"背景粒子/光轨/3D文字飞入/glitch/whip-pan转场"这类非主体特效，再炫也不崩**。原因：背景特效不要求主体保真，模型可自由发挥；只要主体人物保持小幅动作（挥手/举球/摆肩），满屏背景特效照样成。→ 带货宣传片（产品+炫背景特效+文字飞入+快闪卡点）这种"高燃宣传片"形态是可做的，前提是产品/人物主体动作克制、特效都放背景层。
**这条路线换来的代价（无法用提示词补）**：运镜、每个动作的快慢、卡点节奏、单镜时长**全部由 Seedance 自定，你插不上手**——想"这里慢一点/这里推近一下"做不到。要逐镜级精确控制（卡 BGM 节拍、指定运镜、慢镜头气场）就回逐镜路线。**选型：要快要便宜要连贯一致 → 演绎；要精确控制每一拍 → 逐镜。**

**⛔ 硬边界：有"瞬间状态突变"的剧情，演绎路线做不了（萌宠剧情项目实测确认）**。演绎靠**插值补间**把相邻两格平滑过渡——它不理解"两格之间隔着一次突变"，会把突变摊成渐变。实测（一条"角色钻进窝里变身"的萌宠剧情）：变身前的格→变身后的格，模型让"变身后的服饰"**提前渐渐出现**，变成"穿好装备才进窝"，变身因果彻底倒置。凡涉及**变身/换装/物体出现消失/使用前后对比/任何需要"切一刀"才成立的突变**，演绎必穿帮——因为它的"平滑"本质和"突变"叙事天生冲突。逐镜路线靠**黑场/硬切切断插值**藏住突变（如格3→0.5s黑场→格4），演绎一气呵成塞不进黑场，躲不掉。
→ 决策：剧情含突变拍 = 回逐镜（或：突变前后拆成两条独立演绎，中间手动黑场硬切拼接）。纯连续动作的剧情（滑冰、走秀、萌宠日常、无变身的故事）才是演绎甜区。

**⭐ 必须写"格间衔接动作"，不能只写每格的静态状态（萌宠剧情项目实测连犯两次）**：演绎按格演，但**相邻两格之间的位移/过渡动作如果不写，模型会直接跳过**——状态A 直接蹦到状态B，中间过程凭空消失（实测：上一格角色还在 A 处、下一格已在 B 处且任务已完成，"起身→跑过去→伸手完成动作"整段过程被跳掉，结果道具像凭空变出来）。对策：每格除了"到达后的状态"，还要写**"如何从上一格到达本格"的衔接动作**——把"从 A 起身、跑到 B、做出 X 动作"逐拍写出来。要藏的过程（如变身）用遮挡物空窗藏；要展示的过程（如完成某动作）必须逐拍写出来。审片同理：不光看每格状态对不对，要看**格与格之间的过程有没有丢**。

**⭐ 必须给每格分配时间码，动作多的格子单独留足秒数（萌宠剧情项目实测）**：不写时间分配，模型自己决定快慢，**剧情多/时间不够的格子会被它加速跑过去**（高潮动作被一带而过，部分也因没告诉它"这段该占多久"）。对策：开拍前按总时长把秒数分到每格，**动作密度大的格子要明显多给**（如最复杂的那拍给到全片约 1/3 时长），并在该格写"这段X秒要演满、节奏从容、不许加速跳过"。先算账（如总15s 拆成 钩子2+反应1.5+位移2.5+亮相2+核心动作4.5+收尾2.5）再写提示词。

**双图喂法（人物连贯的关键）**：故事板大图之外，**另出一张"角色卡"一起 @ 进去**——故事板管"每格干什么"，角色卡管"人/服装/比例跨格不变"。两张分工："基于故事板生成视频，用故事板做动作规划、用角色卡保持人物面部服装发型比例一致"。
**角色卡的完整规格（人物运动项目实测，越全越稳）**：不只正/侧/背三视图，最好做成**多角度 + 局部细节一图打包**：正面/3-4左/3-4右/侧脸/特写肖像 + 全身正/全身背 + **眼睛特写 + 服装/材质细节特写 + 一张动作姿**。多角度让模型任何机位都有参考，细节特写锁住五官和面料纹理——实测一条 15s 人物运动片里人脸和服装一致性极稳，就靠这种全规格角色卡。做带人物的系列内容，先出这张卡，全集复用。
**角色卡 + 故事板可合并成一张"开发板"**：上半部=角色设计表（多角度+多表情，含每个角色的关键道具姿势），下半部=N格故事板。一次生成让角色锚和分镜同源、一致性更强也更省。适合带人物/IP化角色的剧情带货（数字人系列、IP萌宠）。多角色时每个角色都给一排设计表。

**动态主体的物理段（连续大动作必写）**：滑冰/跳舞/打斗这类运动主体，在提示词里单列一段"物理真实"：主体与环境的接触要有痕迹（冰刀划冰留痕、脚步压地）、跳跃有真实起跳落地、**面料/头发自然滞后于身体**、粒子（雪/尘/水花）对运动有反应。这是 skill.md"物理反馈"从"被作用物体"扩展到"运动主体本身"。
⚠️ **实测结论（一条冰上运动 16 格故事板 → 约 15s 成片，已验）**：这条路线比预期强——优雅舞蹈题材基本走通：①人物跨十几个动作不变脸 ✓；②冰刀贴冰/冰屑/留痕等物理特写最稳（无人脸纯物理是强项）✓；③建立镜头和最终展姿电影级 ✓。**唯一糊的是高速旋转那一拍**——上半身和脸被运动模糊吃成一团。规律：Seedance 单段扛得住"流畅滑行/展姿/慢转/跳跃"，**扛不住本质必须快的高速旋转**（它对其他动作会聪明地"放慢"绕开崩坏，旋转躲不掉）。所以——
- 故事板里**避免设计高速原地旋转**的格子，换成慢转、展姿、滑行、跳跃，整段演绎成片率很高
- 必须要旋转镜头 → 那一格**单独逐镜重抽**（或后期用别的镜头替换），别为一格放弃整条路线
- 题材适配度：**优雅/舒缓/萌宠 = 这条路线的甜区**；高速对抗/打斗/竞技动作仍需逐镜
- ⚠️ **画幅**：故事板演绎默认出 **16:9 横屏**（实测这条就是 1920×1080）——做竖屏带货要么在故事板提示词里强制写每格 9:16 且大图竖版、要么后期裁，别直接发横屏到抖音/TikTok
