# 口播驱动型视频（SRT 声画同步，Veo / Sora / 可灵原生人声）

主文件 §3.7 的完整版。适用：真人口播带货、数字人讲解——**口播是骨架，画面跟着台词走**，与默认的"BGM + 剪映后期配音"工作流互补。当用户提供口播文案 / SRT / 配音音频，或要做"边说边演示"的对口型视频时启用。

**分段规则（按 SRT 时间轴切，不按画面切）**
1. 视频从 0 秒起算，不从口播开始算（口播 4 秒才进，第一段也是 00:00 起）
2. 每段 ≤15 秒，按**视频时长**算，不按口播时长算
3. 段尾必须说完**整句**，绝不在一句话中间切断
4. 小数秒取整：会压到口播就向后取整——口播完整 > 严格四舍五入
5. 1–3 秒的无口播空档并入前后段当过渡画面，不单独切段
6. **语义完整优先于时间硬切**：每段形成一个小主题（如：痛点引入 → 用法成分 → 功效演示 → 促单 CTA），别为凑时长把不相干内容塞同段

**每段提示词的音频模块（可抄格式）**
- 音色全片统一，用英文标签写死：`Voice Profile: [Female, 25-30, warm clear tone, moderate pace, no background noise]`
- 台词逐句带时间戳：`Timed Voiceover: [00:00-00:02] "第一句" -> [00:03-00:05] "第二句". Lip-sync to timestamps.`
- 末尾必带：`NO BGM, no background music. Pure voice and ambient foley only.`——生成只要人声 + 环境音（脚步、按压、水声），BGM 永远后期加（同"关原声"原则）
- 每段都完整携带同一份风格设定与产品约束（铁律 2），跨段才不变样

**与铁律的兼容**：口播镜头能撑到 10–15 秒，是因为人物主要在对镜说话、肢体动作少——**每段仍只安排 1–2 个动作节拍**（如"举起产品 → 挤出展示"），不要逐秒排动作，那是另一种塞节拍翻车。
**段内多拍的写法**：写成显式序列"[动作1：举起产品至胸前] → [动作2：拇指翻开盖子]"，**不要写叙事过渡句**（"画面从手部缓缓过渡到产品特写"）——过渡句等于把怎么压缩、跳哪一拍的决定权交还给模型。

**高能口播的情绪三拍递进**（算账型/痛点型口播带货可借鉴）：①**质问/戳痛**——一句话直戳观众痛处（"还在为X发愁？"）；②**反常识钩子**——给个"反直觉但有道理"的点把人勾住；③**情绪顶点+行动号召**——情绪拉满 + 明确 CTA。真人对镜头口播按这条曲线递进，比平铺直叙有张力。题材选择和合规尺度由用户按目标平台判断，助手不主观替用户设限。
